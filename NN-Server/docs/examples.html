1. Image Classification (CNN)

This example uses a CNN model trained on the ISIC skin lesion dataset (11 classes) with NN-CLI.

Step 1 — Train the model with NN-CLI

# See NN-CLI documentation for training details
./NN-CLI --config isic_cnn_config.json --samples train/samples.json --output train/output/

Step 2 — Create config.json

{
  "model": "train/output/checkpoint_E-150_L-0.029486.json",
  "port": 8080,
  "poolSize": 4
}

Step 3 — Start NN-Server

./NN-Server config.json

# Output:
# NN-Server starting...
#   Config:    config.json
#   Model:     train/output/checkpoint_E-150_L-0.029486.json
#   Port:      8080
#   Pool size: 4
#   Output:    vector (JSON)
#
# Loading model 1/4...
# Loading model 2/4...
# Loading model 3/4...
# Loading model 4/4...
# Server listening on port 8080
# Ready to accept requests.

Step 4 — Classify an image

curl -X POST http://localhost:8080/predict \
     -H "Content-Type: image/jpeg" \
     --data-binary @skin_lesion.jpg

Response

{
  "output": [
    0.0023,   // class 0
    0.0011,   // class 1
    0.0008,   // class 2
    0.9412,   // class 3 — highest probability
    0.0102,   // class 4
    0.0031,   // class 5
    0.0045,   // class 6
    0.0198,   // class 7
    0.0021,   // class 8
    0.0098,   // class 9
    0.0051    // class 10
  ]
}

The model predicts class 3 with 94.1% confidence. The image is automatically resized to the model's input shape (3×450×600 for this model).

2. JSON Input Prediction (ANN)

For dense (ANN) models or when you want to provide pre-processed input directly:

config.json

{
  "model": "ann_model.json"
}

Start the server

./NN-Server config.json

Send a prediction request

curl -X POST http://localhost:8080/predict \
     -H "Content-Type: application/json" \
     -d '{"input": [0.1, 0.5, 0.9, 0.3, 0.7]}'

Response

{
  "output": [0.23, 0.77]
}

3. Concurrent Requests

NN-Server handles multiple requests in parallel up to NN_SERVER_POOL_SIZE. Here's a bash script that sends 10 requests concurrently:

#!/bin/bash
# Send 10 concurrent classification requests

for i in $(seq 1 10); do
  curl -s -X POST http://localhost:8080/predict \
       -H "Content-Type: image/jpeg" \
       --data-binary @image_$i.jpg \
       -o result_$i.json &
done

# Wait for all background jobs
wait

# Check results
for i in $(seq 1 10); do
  echo "Image $i:" $(cat result_$i.json)
done
Pool size vs concurrency: If you send more concurrent requests than the pool size, the excess requests will block until a Core becomes available. They will not be rejected — just queued. For best throughput, set "poolSize" in your config.json to match your expected peak concurrency.

4. Image-to-Image Output

For models that produce image output (segmentation masks, super-resolution, etc.), the server automatically detects the "outputType": "image" field from the model file and uses the required "outputShape" to reconstruct the output as a PNG image.

config.json

{
  "model": "segmentation_model.json"
}

./NN-Server config.json

# Send an image and save the output PNG
curl -X POST http://localhost:8080/predict \
     -H "Content-Type: image/jpeg" \
     --data-binary @input.jpg \
     -o segmentation_output.png

The response is a raw PNG image with the specified shape. The output vector values are clamped to [0, 1] and scaled to [0, 255].

5. Health Monitoring

Use the /health endpoint to integrate with load balancers, Docker health checks, or monitoring tools:

Docker health check

HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD curl -f http://localhost:8080/health || exit 1

Simple uptime script

#!/bin/bash
while true; do
  STATUS=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/health)
  if [ "$STATUS" != "200" ]; then
    echo "$(date): NN-Server is DOWN (status: $STATUS)"
  fi
  sleep 60
done
