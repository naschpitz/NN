NN — Neural Network Monorepo

A C++17 + Qt + OpenCL neural-network monorepo containing four components: ANN, CNN, NN-CLI, and NN-Server. The components that used to live in separate repositories now live together in one git repository.

📦 Monorepo

Four components, one repository — simpler dependency management

⚡ GPU + CPU

OpenCL-accelerated GPU backends alongside multi-threaded CPU

🧩 Modular

CMake build toggles: BUILD_ANN, BUILD_CNN, BUILD_NN_CLI, BUILD_NN_SERVER

📐 Static HTML Docs

Hand-written HTML documentation with a shared theme, LaTeX via MathJax

Components

ANN

Library

Feedforward artificial neural network library with CPU and GPU backends. Template-based, supports ReLU/Sigmoid/Tanh activations, multi-threaded training, and multi-GPU via OpenCL.

Explore ANN docs →

CNN

Library

Convolutional neural network library with Conv2D, pooling, normalization, residual layers, and flatten. Depends on ANN for dense layers. GPU backends via OpenCL with im2col+GEMM.

Explore CNN docs →

NN-CLI

Application

Command-line interface for training, prediction, testing, and calibration. JSON configuration, CPU/GPU device selection, MNIST support, and GPU image augmentation.

Explore NN-CLI docs →

NN-Server

Application

HTTP inference server built on Civetweb. Thread-safe CorePool manages ANN/CNN core instances. REST API for /health and /predict endpoints with JSON I/O.

Explore NN-Server docs →

Clone & Setup

git clone https://github.com/naschpitz/NN.git && cd NN
    # Initialize OpenCLWrapper submodules
    git submodule update --init --recursive

Build

Each component builds into its own build/ directory. A component is built standalone — it pulls its dependencies via add_subdirectory (CNN → ANN → extern/OpenCLWrapper), so building NN-CLI or NN-Server transitively builds the libraries they need.

Build everything — build.sh

# Static Qt6
    ./build.sh static

    # Shared Qt6
    ./build.sh shared

build.sh builds ANN, CNN, NN-CLI and NN-Server, each into its own build/. It reads the Qt6 kit paths from the repo-root CMakeUserPresets.json (git-ignored, machine-specific) and copies that master into each component before building.

Create CMakeUserPresets.json at the repo root with your Qt6 kit paths:

{
      "version": 3,
      "cmakeMinimumRequired": { "major": 3, "minor": 21 },
      "configurePresets": [
        { "name": "static", "binaryDir": "${sourceDir}/build",
          "cacheVariables": { "CMAKE_PREFIX_PATH": "/path/to/Qt6-static", "CMAKE_BUILD_TYPE": "Release" } },
        { "name": "shared", "binaryDir": "${sourceDir}/build",
          "cacheVariables": { "CMAKE_PREFIX_PATH": "/path/to/Qt6-shared", "CMAKE_BUILD_TYPE": "Release" } }
      ],
      "buildPresets": [
        { "name": "static", "configurePreset": "static" },
        { "name": "shared", "configurePreset": "shared" }
      ]
    }
This file is gitignored and machine-specific — keep it local. After creating it, run ./build.sh static or ./build.sh shared to build all components. For single-component builds with a custom Qt6, pass -DCMAKE_PREFIX_PATH=/path/to/Qt6 directly to cmake.

Build a single component

If Qt6 is on the default paths (distro/Homebrew):

# Build just the CLI
    cd NN-CLI && cmake -B build && cmake --build build -j"$(nproc)"

    # Build just the server
    cd NN-Server && cmake -B build && cmake --build build -j"$(nproc)"

For a custom Qt6 path, pass -DCMAKE_PREFIX_PATH=/path/to/Qt6 or use cmake --preset static after running build.sh once.

Unified build from repo root

cmake -B build && cmake --build build -j$(nproc)

    # Toggle individual components
    cmake -B build -DBUILD_NN_SERVER=OFF -DBUILD_NN_CLI=ON && cmake --build build -j$(nproc)

The root CMakeLists.txt provides BUILD_ANN, BUILD_CNN, BUILD_NN_CLI, and BUILD_NN_SERVER toggles. Dependencies are validated — CNN requires ANN, NN-CLI requires both, etc.
