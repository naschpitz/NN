Endpoints

NN-Server exposes a minimal REST API for health checking and running predictions. All endpoints communicate over plain HTTP.

2. GET /health

Returns a simple health check confirming the server is running. No request body is required.

Request

GET /health HTTP/1.1
Host: localhost:8080

Response — 200 OK

Content-Type: application/json

{
  "status": "ok"
}

curl example

curl http://localhost:8080/health

3. POST /predict

Run inference on the loaded model. The input format is determined by the Content-Type header.

JSON Input — Content-Type: application/json

Send a flat array of floats matching the model's input size.

Request

POST /predict HTTP/1.1
Content-Type: application/json

{
  "input": [0.1, 0.5, 0.9, 0.3]
}

Response — 200 OK (JSON output)

Content-Type: application/json

{
  "output": [0.02, 0.85, 0.01, 0.03, 0.01, 0.01, 0.02, 0.01, 0.03, 0.01, 0.00],
  "logits": [-2.10, 1.65, -2.93, -1.85, -2.84, -2.91, -2.20, -2.95, -1.83, -2.97, -3.40]
}

curl example

curl -X POST http://localhost:8080/predict \
     -H "Content-Type: application/json" \
     -d '{"input": [0.1, 0.5, 0.9, 0.3]}'
Input size: For ANN models, the array length must match the first layer's numNeurons. For CNN models, it must equal C × H × W from the model's inputShape.
output vs logits: output holds the post-activation values of the last layer (e.g. softmax probabilities). logits holds the pre-activation values z that feed the final activation. Softmax is shift-invariant — softmax(z) = softmax(z + c) — so it discards magnitude information; two very different logit vectors can yield the same probabilities. Exposing logits lets clients compute calibration / out-of-distribution detection scores: max-logit max(zᵢ), logit-norm ‖z‖, and the free-energy score E(x) = −log Σ exp(zᵢ) (lower = more in-distribution). Argmax of logits always equals argmax of softmax output, so existing class-prediction code keeps working.

Image Input — Content-Type: image/*

Send raw image bytes (JPEG, PNG, BMP, etc.) directly. The server decodes the image, resizes it to the model's inputShape, normalises pixel values to [0, 1], and converts to NCHW layout.

Request

POST /predict HTTP/1.1
Content-Type: image/jpeg
Content-Length: 32451

<raw JPEG bytes>

Response — 200 OK (JSON output)

Content-Type: application/json

{
  "output": [0.02, 0.85, 0.01, 0.12],
  "logits": [-2.10, 1.65, -2.93, -0.16]
}

Response — 200 OK (Image output)

When the model's "outputType" is "image", the response is a PNG image instead. Logits are not emitted for image-output models (they aren't meaningful in that context).

Content-Type: image/png

<raw PNG bytes>

curl example

curl -X POST http://localhost:8080/predict \
     -H "Content-Type: image/jpeg" \
     --data-binary @photo.jpg
Image input requires a model with a defined inputShape. This works for both CNN and ANN models. For ANN models, the image is decoded, resized to the inputShape, and flattened into a 1D vector (e.g., a 1×28×28 MNIST image becomes a 784-element input).

4. Other Routes (404)

Any request that is not GET /health or POST /predict returns a 404 error — including GET /predict, POST /health, and all other method/path combinations.

Content-Type: application/json

{
  "error": "Not found. Use POST /predict"
}
