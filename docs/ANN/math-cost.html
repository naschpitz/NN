Cost Functions

The cost function measures how far the network's prediction is from the expected output. Training minimizes the cost (also called the loss — the scalar value produced by the cost function) by adjusting weights and biases.

Squared Difference (MSE)

ANN_Worker.cpp — calculateLoss()

Cost

\[ C = \frac{1}{N} \sum_{i=1}^{N} w_i \cdot (a_i - y_i)^2 \]

Gradient (for backpropagation)

\[ \frac{\partial C}{\partial a_j} = 2 \cdot w_j \cdot (a_j - y_j) \]

Where \(a_j\) is the predicted output, \(y_j\) is the expected output, and \(w_j\) is the per-output weight (defaults to 1). Suitable for regression tasks.

Cross-Entropy

ANN_Worker.cpp — calculateLoss()

Cost

\[ C = -\sum_{i=1}^{N} w_i \cdot y_i \cdot \log(a_i) \]

Gradient (for backpropagation)

\[ \frac{\partial C}{\partial a_j} = -\frac{w_j \cdot y_j}{a_j} \]

Where \(a_j\) is clamped to a minimum of \(10^{-7}\) to avoid division by zero. Cross-entropy is the standard cost function for classification tasks, typically paired with softmax on the output layer.

Weighted Variants

Both cost functions support per-output weights \(w_i\) via the costFunctionConfig.weights array. This is useful for class-imbalanced datasets — assigning higher weight to underrepresented classes increases their influence on the gradient.

When no weights are specified, all \(w_i = 1\).

Config ValueEnumDescription
"squaredDifference"SQUARED_DIFFERENCEMSE with uniform weights
"weightedSquaredDifference"WEIGHTED_SQUARED_DIFFERENCEMSE with per-output weights
"crossEntropy"CROSS_ENTROPYCross-entropy (supports weights)

Implementation

ANN_CoreCPUWorker.cpp — calc_dCost_dActv()

The output layer gradient is computed per-neuron based on the cost function type:

switch (this->costFunctionConfig.type) {
  case CostFunctionType::CROSS_ENTROPY: {
    const T epsilon = 1e-7;
    T pred = std::max(this->actvs[l][j], epsilon);
    return -weight * output[j] / pred;
  }
  case CostFunctionType::SQUARED_DIFFERENCE:
  default:
    return 2 * weight * (this->actvs[l][j] - output[j]);
}
