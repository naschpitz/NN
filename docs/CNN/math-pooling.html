Pooling Backpropagation

Pooling layers reduce spatial dimensions by aggregating values. During backpropagation, gradients must be routed back to the input elements that contributed to the pooling operation.

On this page

  1. Max Pooling
  2. Average Pooling

Max Pooling

In max pooling, only the maximum value in each pool window contributes to the output. During backpropagation, the gradient flows entirely to the position of the maximum value:

\[ \frac{\partial C}{\partial X[c,i_h,i_w]} = \begin{cases} \frac{\partial C}{\partial O[c,o_h,o_w]} & \text{if } (i_h, i_w) = \text{argmax} \\ 0 & \text{otherwise} \end{cases} \]

Average Pooling

In average pooling, the output is the mean of all values in the pool window. The gradient is distributed equally among all input elements:

\[ \frac{\partial C}{\partial X[c,i_h,i_w]} = \frac{1}{P_H \cdot P_W} \cdot \frac{\partial C}{\partial O[c,o_h,o_w]} \]

where \(P_H\) and \(P_W\) are the pool window dimensions.

Code: Pool<T>::propagate(input, config, maxIndices) in CNN_Pool.cpp implements both modes. For max pooling, it stores the flat index of each maximum in maxIndices. Pool<T>::backpropagate() routes the gradient to dInput.data[maxIdx] (max) or distributes dOutVal / count evenly (avg). On GPU: calculate_maxpool records indices (opencl/CNN_Propagate.cpp.cl), calculate_dCost_dMaxpool routes gradients (opencl/CNN_Backpropagate.cpp.cl).
Max pooling is more commonly used in practice because it preserves sharp features and provides implicit regularization. Average pooling is sometimes used in the final layers for global feature aggregation.
