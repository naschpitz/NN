CNN + ANN Pipeline

The CNN library doesn't work in isolation — it forms the feature extraction front-end of a complete neural network pipeline. Convolutional layers extract spatial features from images, then the ANN library's dense layers perform the final classification or regression. This page explains how the two libraries work together, how data flows between them, and what happens at the critical handoff point.

Pipeline Overview

The complete inference pipeline chains CNN layers (convolutional, activation, pooling, normalization, residual, global pooling) with ANN dense layers. Data flows as a 3D tensor through CNN, gets flattened to 1D, then flows through ANN's fully-connected layers to produce the final output.

CNN Phase (Feature Extraction)

The CNN phase processes the raw input image (a 3D tensor with shape C×H×W) through a sequence of layers that progressively transform it into more abstract feature representations:

  1. Conv2D — applies learned filters to detect local patterns (edges, textures, shapes). Outputs a 3D tensor with potentially different channel count and spatial dimensions.
  2. InstanceNorm (optional but recommended) — normalizes the activations per channel, stabilizing training and enabling higher learning rates.
  3. ReLU — introduces non-linearity by zeroing out negative values. Without this, the entire network would be a linear transformation.
  4. Pool — reduces spatial dimensions (e.g., 28×28 → 14×14), decreasing computation and providing translation invariance.
  5. Residual blocks (optional) — wrap some layers with skip connections, enabling deeper architectures without gradient vanishing.
  6. Global pooling (optional) — collapses spatial dimensions to 1×1, dramatically reducing the parameter count at the CNN→ANN boundary.

These layers are typically repeated in blocks (Conv→Norm→ReLU→Pool), with each block increasing the channel count while decreasing spatial dimensions. Early layers detect simple features (edges, color gradients); deeper layers combine these into complex patterns (textures, object parts, whole objects).

The Handoff: Flatten

The Flatten layer is the critical junction between CNN and ANN. It takes the final 3D tensor from the CNN phase and reshapes it into a 1D vector that dense layers can process:

\[ \text{3D tensor } (C, H, W) \xrightarrow{\text{flatten}} \text{1D vector of } C \times H \times W \text{ elements} \]

On the CPU backend, this is a zero-copy operation (the data is already stored as a flat vector). On the GPU backend, the flattened data is copied from GPU memory to CPU memory at this point, since ANN dense layers run on the CPU.

Size matters: The flattened vector size determines the first dense layer's input dimension. Without global pooling, this can be huge (e.g., 512 channels × 4×4 spatial = 8,192 elements). With GlobalAvgPool, it's just the channel count (e.g., 512). This is why modern architectures almost always use global pooling before flatten.

ANN Phase (Classification)

The ANN library receives the flattened vector and processes it through one or more fully-connected (dense) layers. Each dense layer:

  1. Linear transformation — multiplies the input by a weight matrix and adds biases: \(z = Wx + b\)
  2. Activation function — applies a non-linearity (ReLU, sigmoid, softmax, etc.)

The final layer typically uses:

  • Softmax for multi-class classification — outputs probabilities that sum to 1
  • Sigmoid for binary classification — outputs a value in [0, 1]
  • Linear (no activation) for regression — outputs unbounded values

The ANN library handles its own weight management, gradient computation, and optimizer updates internally. The CNN library treats ANN as a black box — it sends the flattened vector in, gets the prediction out.

Backward Pipeline

During training, the backward pipeline runs in reverse. After computing the loss (comparing predicted output to expected output):

  1. ANN backpropagation — ANN computes gradients through its dense layers and returns the gradient with respect to its input (the flattened vector).
  2. Unflatten — the 1D gradient is reshaped back to 3D (on GPU, it's copied from CPU back to GPU memory).
  3. CNN backpropagation — gradients flow backward through the CNN layers in reverse order. Each layer computes the gradient with respect to its input (passed to the previous layer) and the gradient with respect to its parameters (accumulated for the optimizer).

The key insight is that both CNN and ANN gradients must be correct for learning to work. If either side has a bug, the entire pipeline fails to converge. The handoff at the flatten/unflatten boundary must preserve the gradient values exactly.

Debugging tip: If training produces NaN or Inf values, the problem usually originates in one specific layer and propagates through the pipeline. Check the loss values at each epoch — a sudden jump to infinity often indicates an exploding gradient in one of the early Conv2D layers. Adding InstanceNorm typically fixes this.
