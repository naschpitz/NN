Training Pipeline

Training is the process by which the CNN learns to transform input images into correct predictions. It works by repeatedly showing the network training examples, measuring how wrong its predictions are (the loss), and adjusting the network's parameters to reduce that error. This page walks through every step of the training loop in detail.

Why Neural Networks Learn

A neural network starts as a meaningless pile of random numbers — thousands of filter weights, biases, normalization parameters, and dense layer weights, all initialized to small random values. Its initial predictions are pure noise. Yet after training, those same weights somehow encode sophisticated knowledge about how to recognize objects, classify images, or make predictions. How does this happen?

The answer rests on a beautifully simple loop:

The Learning Loop

  1. Forward pass — feed an image through the network and get a prediction. With random weights, the prediction is garbage.
  2. Measure the error — compare the prediction to the known correct answer using a loss function. The loss is a single number measuring "how wrong" the prediction is. Higher = worse.
  3. Trace the blame — use backpropagation to compute, for every single weight in the entire network, how much that weight contributed to the error. The result is a gradient: a number saying "if this weight increased by a tiny amount, the loss would change by this much."
  4. Adjust the weights — nudge each weight in the direction that reduces the loss. If increasing a weight would increase the loss (positive gradient), decrease it slightly. If increasing it would decrease the loss (negative gradient), increase it slightly.
  5. Repeat — do this for every training image, epoch after epoch. Each iteration makes the weights slightly better. Over thousands of iterations, the weights converge to values that produce accurate predictions.

This process is called gradient descent — you're "descending" the loss landscape by following the gradient downhill. It's the same principle as a ball rolling downhill: it naturally moves toward the lowest point (minimum loss).

The remarkable thing is that nobody tells the network what to look for. Nobody programs "look for edges in layer 1 and textures in layer 2." The network discovers these representations on its own because they happen to be the weight values that minimize the loss on the training data. The loss function (e.g., "penalize wrong classifications") is the only supervision the network receives.

Why does backpropagation work across so many layers? Backpropagation applies the chain rule from calculus. Each layer knows its own local gradient (how its output changes with respect to its input and parameters). By multiplying these local gradients backward through the network, we get the gradient of the loss with respect to any weight, no matter how many layers deep. It's like tracing responsibility through a chain of events: "the loss was high because layer 5's output was wrong, which happened because layer 4's output was wrong, which happened because layer 3's filter weight was too large."

Key Concepts

With the learning loop in mind, here are the essential terms used throughout the training pipeline:

  • Epoch — one complete pass through the entire training dataset. If you have 1,000 images and train for 50 epochs, the network sees each image 50 times (in different random orders). Each epoch gives the weights another chance to improve.
  • Sample — a single input-output pair: an image (input) and its correct label (expected output). The network learns by comparing its prediction to the expected output.
  • Loss — a number measuring how wrong the prediction is. Lower loss = better predictions. Common loss functions include Mean Squared Error (MSE) for regression and Cross-Entropy for classification. The loss is the signal that drives all learning.
  • Gradient — for each weight, the gradient says "how much the loss would change if this weight changed by a tiny amount." Positive gradient means increasing the weight would increase the loss (bad); negative gradient means increasing the weight would decrease the loss (good). Computed via backpropagation.
  • Learning rate — controls how big a step the optimizer takes in the gradient direction. Too high → weights overshoot and training diverges. Too low → training is painfully slow. Typical values: 0.001 to 0.01.
  • Shuffling — randomizing the order of training samples each epoch. This prevents the network from memorizing patterns based on sample ordering (e.g., "all cats come first") and improves generalization to new data.

Training Loop Overview

for epoch in [0, numEpochs):
  shuffle(samples)

  for batch of batchSize samples:
    resetCNNAccumulators()
    for sample in batch:
      // 1. Propagate through CNN (store intermediates)
      cnnOut = propagateCNN(sample.input, true, &intermediates, &poolMaxIndices)

      // 2. Flatten and propagate through ANN
      flatInput = Flatten::propagate(cnnOut)
      annOutput = annCore->predict(flatInput)

      // 3. Compute loss (MSE)
      loss = MSE(annOutput, sample.output)

      // 4. Backprop through ANN
      dANNInput = annCore->backpropagate(dLoss)

      // 5. Unflatten and backprop through CNN
      dCNNOut = Flatten::backpropagate(dANNInput)
      backpropagateCNN(dCNNOut, intermediates, poolMaxIndices, dFilters, dBiases)

      // 6. Accumulate gradients
      accumulateCNNGradients(dFilters, dBiases)

    // 7. Update parameters (SGD, per batch)
    updateCNNParameters(batchSize)
    annCore->updateParameters(batchSize)

Detailed Steps

1. Propagation (CNN)

Input flows through CNN layers in order. Intermediates (layer inputs) are stored for backpropagation. Pool layer max indices are recorded for gradient routing.

Tensor3D<T> current = input;
for (const auto& layerConfig : layersConfig.cnnLayers) {
  switch (layerConfig.type) {
    case LayerType::CONV:
      intermediates.push_back(current);
      current = Conv2D<T>::propagate(current, config, params);
      break;
    case LayerType::RELU:
      intermediates.push_back(current);
      current = ReLU<T>::propagate(current);
      break;
    // ... Pool, Flatten
  }
}

2. Propagation (ANN)

Flattened CNN output becomes input to ANN dense layers. ANN handles its own propagation and gradient computation.

Tensor1D<T> flatInput = Flatten<T>::propagate(cnnOut);
ANN::Input<T> annInput(flatInput.begin(), flatInput.end());
ANN::Output<T> annOutput = annCore->predict(annInput);

3. Loss Computation

Mean Squared Error (MSE) between predicted and expected output.

T calculateLoss(const Output<T>& predicted, const Output<T>& expected) {
  T loss = 0;
  for (ulong i = 0; i < predicted.size(); i++) {
    T diff = predicted[i] - expected[i];
    loss += diff * diff;
  }
  return loss / predicted.size();
}

4. Backpropagation (ANN)

ANN computes gradient of loss w.r.t. its input (the flattened CNN output). ANN also updates its own parameters internally.

// ANN backprop returns gradient w.r.t. its input
ANN::Output<T> dANNInput = annCore->backpropagate(dLoss);

5. Backpropagation (CNN)

Gradient flows through CNN layers in reverse order. Each layer computes gradient w.r.t. its input and accumulates gradient w.r.t. its parameters.

// Unflatten gradient from ANN
Tensor3D<T> dCNNOut = Flatten<T>::backpropagate(dANNInput);

// Backprop through CNN layers in reverse
Tensor3D<T> dCurrent = dCNNOut;
for (int i = cnnLayers.size() - 1; i >= 0; i--) {
  switch (cnnLayers[i].type) {
    case LayerType::CONV:
      dCurrent = Conv2D<T>::backpropagate(
        dCurrent, intermediates[i], config, params, dFilters, dBiases);
      break;
    // ... ReLU, Pool
  }
}

6. Gradient Accumulation

Gradients from each sample are accumulated across the epoch. This enables batch-like updates without storing all samples in memory simultaneously.

7. Parameter Update (SGD)

After processing all samples in a mini-batch, accumulated gradients are used to update parameters via Stochastic Gradient Descent.

void updateCNNParameters(ulong numSamples) {
  T learningRate = trainingConfig.learningRate;
  for (ulong i = 0; i < parameters.convParams.size(); i++) {
    for (ulong j = 0; j < parameters.convParams[i].filters.size(); j++) {
      parameters.convParams[i].filters[j] -=
        learningRate * accumDConvFilters[i][j] / numSamples;
    }
  }
}

What Each Layer Learns

During training, different layers develop different specializations. This happens automatically — the loss signal propagates backward through the chain, and each layer adjusts its weights to best serve its role in the pipeline:

LayerWhat it learnsHow
Conv2D (early) Simple, local patterns: edges, color gradients, spots Filter weights converge to small patterns (like edge detectors) that are broadly useful across many images. These patterns are often visually interpretable.
Conv2D (deep) Complex, compositional patterns: textures, object parts, shapes Deep filters operate on the feature maps from earlier layers, combining simple features into complex ones. A "face detector" filter might activate when it sees two eye-like patches above a mouth-like patch.
InstanceNorm Optimal scale (γ) and offset (β) for each feature channel Some channels work best with amplified activations (high γ), others with suppressed activations. The network learns these per-channel scaling factors to optimize the activation distribution for the subsequent ReLU.
Residual (projection) How to map feature channels when dimensions change The 1×1 projection learns which combinations of input channels should map to which output channels in the skip connection.
Dense layers (ANN) How to map the extracted features to output classes Dense layer weights learn which combinations of features (from the final CNN output) are indicative of each class. For cat vs. dog classification, specific feature combinations activate for cats and different ones for dogs.

Layers without learnable parameters (ReLU, Pool, GlobalAvgPool, Flatten) don't "learn" anything — they apply fixed transformations. But they are essential because they shape the data flow: ReLU introduces the non-linearity that makes learning complex patterns possible, Pool provides translation invariance and downsampling, and GlobalAvgPool eliminates spatial information to produce a compact representation.

The feature hierarchy: The progression from simple → complex features is not programmed; it emerges naturally from the network structure. Early layers see small spatial regions (3×3 or 5×5 pixels) and can only detect simple patterns. Deeper layers see larger effective regions (because each layer's receptive field builds on the previous layer's) and can detect correspondingly more complex patterns. This hierarchical feature extraction is the fundamental reason CNNs work so well for vision tasks.

Convergence & Monitoring

Training is monitored through the loss value reported at each epoch. A healthy training run shows:

  • Steady decrease — the loss should generally decrease over epochs, though small fluctuations are normal.
  • Diminishing returns — the loss decreases rapidly at first, then more slowly. This is expected — easy patterns are learned first.
  • Plateau — eventually the loss stops decreasing significantly. This usually means the model has learned what it can from the data. Additional epochs provide no benefit.

Warning signs during training:

SymptomLikely CauseFix
Loss increases or oscillates wildlyLearning rate too highReduce learning rate by 10×
Loss decreases extremely slowlyLearning rate too lowIncrease learning rate by 2-5×
Loss becomes NaN or InfNumerical instability (exploding gradients)Add InstanceNorm layers, reduce learning rate
Training loss is good but test loss is badOverfittingMore training data, add regularization, simplify network
Loss stays constant from epoch 1Dead network (all ReLUs saturated)Check weight initialization, reduce learning rate
Practical tip: Start with a learning rate of 0.001 and adjust from there. If using InstanceNorm, you can often use higher learning rates (0.01 or even 0.1). Monitor the loss for the first few epochs — if it's not decreasing, the learning rate is likely wrong.
