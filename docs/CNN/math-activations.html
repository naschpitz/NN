Activation Functions

Activation functions introduce non-linearity into the network. Without them, stacking multiple layers would be equivalent to a single linear transformation. The library supports four activation types via ANN::ActvFuncType: RELU, SIGMOID, TANH, and SOFTMAX.

On this page

  1. ReLU
  2. Sigmoid
  3. Tanh
  4. Softmax

ReLU

The Rectified Linear Unit (ReLU) activation function is defined as \(y = \max(0, x)\). During backpropagation, the gradient is zero for negative inputs and passes through unchanged for positive inputs:

\[ \frac{\partial C}{\partial X_i} = \begin{cases} \frac{\partial C}{\partial Y_i} & \text{if } X_i > 0 \\ 0 & \text{otherwise} \end{cases} \]

This can be written compactly as: \(\frac{\partial C}{\partial X_i} = \mathbb{1}_{X_i > 0} \cdot \frac{\partial C}{\partial Y_i}\), where \(\mathbb{1}\) is the indicator function.

Code: ReLU<T>::propagate(input) and ReLU<T>::backpropagate(dOut, input) in CNN_ReLU.hpp. Backpropagation uses the original input to determine where the gradient passes through (input.data[i] > 0). On GPU: calculate_relu kernel in opencl/CNN_Propagate.cpp.cl, calculate_dCost_dRelu in opencl/CNN_Backpropagate.cpp.cl.
ReLU is computationally efficient and helps mitigate the vanishing gradient problem compared to sigmoid or tanh activations. The gradient is either 0 or 1, making the backpropagation straightforward.

Sigmoid

The sigmoid activation squashes values into the range \((0, 1)\):

\[ \text{forward: } \sigma(x) = \frac{1}{1 + e^{-x}} \] \[ \text{backward: } \frac{\partial \sigma}{\partial x} = \sigma(x) \cdot (1 - \sigma(x)) \]
Code: ANN::ActvFunc::calculate(x, ActvFuncType::SIGMOID) in ANN_ActvFunc.hpp. Uses the stored activation value to compute the derivative efficiently.

Tanh

The hyperbolic tangent activation squashes values into \((-1, 1)\):

\[ \text{forward: } \tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}} \] \[ \text{backward: } \frac{\partial \tanh}{\partial x} = 1 - \tanh^2(x) \]
Code: ANN::ActvFunc::calculate(x, ActvFuncType::TANH) in ANN_ActvFunc.hpp. Zero-centered output helps with gradient flow in some architectures.

Softmax

Softmax converts a vector of logits into a probability distribution, used in the output layer for classification:

\[ \text{forward: } s_i = \frac{e^{z_i}}{\sum_j e^{z_j}} \]

The backward pass involves the full Jacobian matrix. For efficiency, the code computes the gradient directly:

\[ \frac{\partial C}{\partial z_i} = s_i \cdot \left( \frac{\partial C}{\partial a_i} - \sum_j s_j \cdot \frac{\partial C}{\partial a_j} \right) \]
For numerical stability, the forward pass first subtracts \(\max(z)\) from all logits before exponentiation. Softmax is a layer-wide activation (applied across all output neurons jointly), unlike ReLU/sigmoid/tanh which operate on individual neurons.
Code: ANN::ActvFunc::calculate(zs, actvs, numNeurons, ActvFuncType::SOFTMAX, ...) in ANN_ActvFunc.hpp. The forward pass computes \(s_i = e^{z_i - \max(z)} / \sum e^{z_j - \max(z)}\). The backward pass computes the Jacobian product \(\partial C/\partial z_i = s_i(\partial C/\partial a_i - \sum_j s_j \cdot \partial C/\partial a_j)\).
