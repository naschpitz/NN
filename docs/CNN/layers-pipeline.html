Layer Pipeline Example

This page provides concrete, step-by-step examples showing how data flows through a sequence of CNN layers. Understanding the shape transformations and parameter counts at each stage is essential for designing effective architectures. We walk through two examples: a simple classification pipeline and a modern architecture using residual blocks and global pooling.

On this page

  1. Simple Classification Pipeline
  2. Modern Architecture Pipeline
  3. Architecture Design Guidelines

Simple Classification Pipeline

This example shows a minimal but complete pipeline for classifying a small grayscale image. Each step shows the shape transformation, the math involved, and the parameter count.

Example: Grayscale Image → Class Prediction

Input: 1 channel, 6×6 image (e.g., grayscale digit)

\[ \text{Shape: } 1 \times 6 \times 6 \]

Step 1: Conv2D (2 filters, 3×3, stride 1, VALID)

Output shape: \(\text{outH} = (6 + 0 - 3) / 1 + 1 = 4\), \(\text{outW} = 4\)

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: \(2 \times 1 \times 3 \times 3 + 2 = 20\) (18 weights + 2 biases)

Step 2: InstanceNorm

Normalize per channel, scale by \(\gamma\), shift by \(\beta\). Shape unchanged

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: \(2 \times 2 = 4\) learnable (\(\gamma, \beta\)) + 4 running stats

Step 3: ReLU

Element-wise \(\max(0, x)\), shape unchanged

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: 0 (no learnable parameters)

Step 4: MaxPool (2×2, stride 2)

Output shape: \(\text{outH} = (4 - 2) / 2 + 1 = 2\), \(\text{outW} = 2\)

\[ \text{Shape: } 2 \times 2 \times 2 \]

Parameters: 0 (no learnable parameters)

Step 5: Flatten

Reshape to 1D vector: \(2 \times 2 \times 2 = 8\) elements

\[ \text{Shape: } 8 \]

Parameters: 0 (no learnable parameters)

The 8-element vector can now be fed to a dense layer for classification.

Modern Architecture Pipeline

This example uses residual blocks, InstanceNorm, and GlobalAvgPool — the modern pattern that enables deeper, better-performing networks.

Example: RGB Image → 10-Class Classification

Input: 3 channels (RGB), 32×32 image

\[ \text{Shape: } 3 \times 32 \times 32 \]

Step 1: Conv2D (32 filters, 3×3, SAME padding)

Initial feature extraction. SAME padding preserves spatial dimensions.

\[ \text{Shape: } 32 \times 32 \times 32 \]

Parameters: \(32 \times 3 \times 3 \times 3 + 32 = 896\)

Step 2: InstanceNorm + ReLU

Normalize and activate. Shape unchanged: \(32 \times 32 \times 32\)

Parameters: 64 learnable (γ, β) + 64 running stats

Step 3: Residual Block (32 → 32, identity shortcut)

Conv2D(32, 3×3, SAME) → InstanceNorm → ReLU, with skip connection. No projection needed since channels stay at 32.

\[ \text{Shape: } 32 \times 32 \times 32 \]

Parameters: \(32 \times 32 \times 3 \times 3 + 32 = 9{,}248\) (conv) + 64 (norm) = 9,312

Step 4: MaxPool (2×2, stride 2)

Spatial downsampling outside the residual block.

\[ \text{Shape: } 32 \times 16 \times 16 \]

Step 5: Residual Block (32 → 64, projection shortcut)

Conv2D(64, 3×3, SAME) → InstanceNorm → ReLU. Channel count increases, so a 1×1 projection is auto-created.

\[ \text{Shape: } 64 \times 16 \times 16 \]

Parameters: \(64 \times 32 \times 3 \times 3 + 64 = 18{,}496\) (conv) + 128 (norm) + \(64 \times 32 + 64 = 2{,}112\) (projection) = 20,736

Step 6: GlobalAvgPool

Collapse each 16×16 feature map to a single value. This is the key — without GlobalAvgPool, flatten would produce \(64 \times 16 \times 16 = 16{,}384\) elements.

\[ \text{Shape: } 64 \times 1 \times 1 \]

Step 7: Flatten

Reshape to 1D vector: just 64 elements.

\[ \text{Shape: } 64 \]

The 64-element vector feeds into ANN dense layers (e.g., 64→32→10) for final classification.

Architecture Design Guidelines

General principles for designing CNN architectures:
  • Increase channels, decrease spatial: As you go deeper, increase the number of filters (32→64→128) while decreasing spatial dimensions with pooling. This maintains roughly constant computation per layer.
  • Use SAME padding inside residual blocks: Residual connections require matching spatial dimensions. Use SAME padding for convolutions inside residual blocks.
  • Pool outside residual blocks: Place MaxPool after RESIDUAL_END, not inside the block. Spatial downsampling inside a residual block breaks the skip connection.
  • Use GlobalAvgPool before flatten: This dramatically reduces the number of dense layer parameters and provides regularization.
  • InstanceNorm before ReLU: Always place normalization before the activation function for best results.
  • Start with fewer channels: Begin with 16 or 32 filters and increase gradually. More initial filters means more parameters and slower training.
Memory Layout: The CNN library uses NCHW (channels-first) layout. When flattening, elements are ordered as: all values in channel 0, then channel 1, etc. This differs from NHWC (channels-last) used by some frameworks like TensorFlow.
