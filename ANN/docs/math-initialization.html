Weight Initialization

Proper weight initialization is critical for training deep networks. Weights that are too large cause exploding activations; weights that are too small cause vanishing gradients. The initialization strategy depends on the activation function.

On this page

  1. He Initialization (ReLU)
  2. Xavier Initialization (Sigmoid / Tanh)
  3. Bias Initialization
  4. Implementation

He Initialization (ReLU)

Used when the layer's activation function is ReLU:

\[ w \sim \mathcal{N}\!\left(0, \;\sqrt{\frac{2}{\text{fan\_in}}}\right) \]

Where \(\text{fan\_in}\) is the number of neurons in the previous layer. The factor of 2 accounts for ReLU zeroing out approximately half of the activations — without it, the variance of activations would shrink by half at each layer.

Xavier Initialization (Sigmoid / Tanh)

Used when the layer's activation function is sigmoid or tanh:

\[ w \sim \mathcal{N}\!\left(0, \;\sqrt{\frac{1}{\text{fan\_in}}}\right) \]

Xavier initialization maintains the variance of activations across layers for symmetric activation functions. Since sigmoid and tanh are approximately linear near zero, the factor of 1 (instead of 2) is appropriate.

Bias Initialization

\[ b = 0 \]

All biases are initialized to zero. This is standard practice — the symmetry-breaking role is handled entirely by the random weight initialization.

Implementation

ANN_CoreCPU.cpp — initializeParameters()

for (ulong l = 1; l < numLayers; l++) {
  ActvFuncType actvFuncType = layer.actvFuncType;
  double stddev;

  if (actvFuncType == ActvFuncType::RELU)
    stddev = std::sqrt(2.0 / fan_in);  // He
  else
    stddev = std::sqrt(1.0 / fan_in);  // Xavier

  std::normal_distribution<double> dist(0.0, stddev);

  for (ulong j = 0; j < numNeurons; j++)
    for (ulong k = 0; k < prevNumNeurons; k++)
      weights[l][j][k] = dist(gen);

  biases[l] = 0;  // All biases initialized to zero
}
Softmax layers use Xavier initialization (the else branch), since softmax is not ReLU. This is appropriate because softmax's input is the raw pre-activation, and Xavier keeps the variance stable for non-ReLU activations.
