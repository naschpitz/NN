Activation Functions

Activation functions introduce non-linearity into the network. Each function has a corresponding derivative used during backpropagation.

ReLU (Rectified Linear Unit)

ANN_ActvFunc.cpp — relu(), drelu()

\[ f(x) = \max(0, x) \qquad\qquad f'(x) = \begin{cases} 1 & \text{if } x > 0 \\ 0 & \text{otherwise} \end{cases} \]

The most widely used activation. Simple, fast, and avoids the vanishing gradient problem for positive values. The derivative is discontinuous at \(x = 0\) but this is handled by defining \(f'(0) = 0\).

Sigmoid

ANN_ActvFunc.cpp — sigmoid(), dsigmoid()

\[ f(x) = \frac{1}{1 + e^{-x}} \qquad\qquad f'(x) = f(x) \cdot (1 - f(x)) \]

Squashes output to the range \((0, 1)\). The derivative is expressed in terms of the function itself, which is computationally convenient. Prone to vanishing gradients for large \(|x|\).

Tanh (Hyperbolic Tangent)

ANN_ActvFunc.cpp — tanh(), dtanh()

\[ f(x) = \tanh(x) \qquad\qquad f'(x) = 1 - \tanh^2(x) \]

Squashes output to the range \((-1, 1)\). Zero-centered, which can help convergence compared to sigmoid. Still suffers from vanishing gradients at saturation.

Softmax

ANN_ActvFunc.hpp — softmax(), dsoftmax()

Forward

Unlike the other activations, softmax operates on the entire layer at once — each output depends on all inputs:

\[ s_j = \frac{e^{z_j - z_{\max}}}{\sum_{i} e^{z_i - z_{\max}}} \]

The subtraction of \(z_{\max}\) is for numerical stability (prevents overflow) and does not change the result. The outputs sum to 1, making softmax suitable for classification (each output represents a probability).

Backward

The Jacobian of softmax is a full matrix, but when combined with the upstream gradient \(\partial L / \partial a_j\), the derivative simplifies to:

\[ \frac{\partial L}{\partial z_j} = s_j \left( \frac{\partial L}{\partial a_j} - \sum_{i} s_i \cdot \frac{\partial L}{\partial a_i} \right) \]

This avoids constructing the full Jacobian matrix. The term \(\sum_i s_i \cdot \partial L / \partial a_i\) is a scalar computed once and reused for all \(j\).

Implementation

ANN_ActvFunc.hpp — ActvFunc::calculate()

All activations are dispatched through a single static method that handles both forward and backward passes:

static void calculate(const T* zs, T* actvs, ulong numNeurons,
                      ActvFuncType type, bool derivative,
                      const T* dCost_dActvs = nullptr,
                      T* dCost_dZs = nullptr);

When derivative = false, it computes the forward activation. When derivative = true, it computes \(\partial L / \partial z\) using the activation's derivative and the upstream gradient dCost_dActvs.
