Instance Normalization

Instance Normalization (IN) normalises each sample independently over its spatial dimensions (H × W) per channel. Unlike Batch Normalization, which computes statistics across a batch of samples, IN computes statistics per individual sample. This makes it independent of batch size and well-suited for architectures that process samples one at a time.

Instance Norm vs Batch Norm

For a feature map with shape \((N, C, H, W)\):

PropertyBatch NormalizationInstance Normalization
Statistics computed over\(N \times H \times W\) (batch + spatial)\(H \times W\) (spatial only)
One mean/var perChannelChannel × Sample
Depends on batch sizeYesNo
Inference behaviourUses running population statsComputes per-sample stats (same as training)
This library uses Instance Normalization. The config layer type is "instancenorm". Because statistics are per-sample, inference uses the same computation as training — no running statistics are needed.

Forward Pass

For each channel \(c\) in a single sample, compute the spatial mean and variance, then normalise and apply learnable scale (\(\gamma\)) and shift (\(\beta\)):

Step 1 — Spatial Mean

\[\mu_c = \frac{1}{H \cdot W} \sum_{h=0}^{H-1} \sum_{w=0}^{W-1} X[c, h, w]\]

Step 2 — Spatial Variance

\[\sigma_c^2 = \frac{1}{H \cdot W} \sum_{h=0}^{H-1} \sum_{w=0}^{W-1} (X[c, h, w] - \mu_c)^2\]

Step 3 — Normalise

\[\hat{X}[c, h, w] = \frac{X[c, h, w] - \mu_c}{\sqrt{\sigma_c^2 + \epsilon}}\]

Step 4 — Scale and Shift

\[Y[c, h, w] = \gamma_c \cdot \hat{X}[c, h, w] + \beta_c\]

where \(\epsilon\) is a small constant (default \(10^{-5}\)) for numerical stability, and \(\gamma_c\), \(\beta_c\) are learnable parameters initialised to 1 and 0 respectively.

Code: InstanceNorm<T>::propagate(input, shape, params, config, training) in CNN_InstanceNorm.cpp. On GPU: calculate_norm_mean, calculate_norm_var, and calculate_norm_propagate kernels in opencl/CNN_InstanceNorm.cpp.cl.

Backward Pass

Given upstream gradient \(\frac{\partial L}{\partial Y}\), we compute gradients for \(\gamma\), \(\beta\), and the input \(X\). Let \(S = H \cdot W\) be the spatial size.

Gamma Gradient

\[\frac{\partial L}{\partial \gamma_c} = \sum_{h,w} \frac{\partial L}{\partial Y[c,h,w]} \cdot \hat{X}[c,h,w]\]

Beta Gradient

\[\frac{\partial L}{\partial \beta_c} = \sum_{h,w} \frac{\partial L}{\partial Y[c,h,w]}\]

Input Gradient

\[\frac{\partial L}{\partial X[c,h,w]} = \frac{\gamma_c}{\sqrt{\sigma_c^2 + \epsilon}} \left( \frac{\partial L}{\partial Y[c,h,w]} - \frac{1}{S} \frac{\partial L}{\partial \beta_c} - \frac{\hat{X}[c,h,w]}{S} \frac{\partial L}{\partial \gamma_c} \right)\]
Code: InstanceNorm<T>::backpropagate(dOut, input, shape, params, config) in CNN_InstanceNorm.cpp. On GPU: calculate_norm_dGamma, calculate_norm_dBeta, and calculate_norm_dInput kernels.

Inference

Unlike Batch Normalization, Instance Normalization uses the same computation during both training and inference. Per-sample spatial statistics are computed on the fly — there is no need for running mean/variance.

The \(\gamma\) and \(\beta\) parameters are the only learned values saved in the trained model. They are stored in the parameters.instancenorm section of the output JSON alongside runningMean and runningVar (which are accumulated but not used during inference).
