GPU Buffer Layout

On this page

Overview

The CNN GPU implementation uses several OpenCL buffers to store activations, gradients, filter weights, biases, and intermediate data. Buffers are allocated once during initialization and reused across training iterations. Most buffers are sized per-batch (one copy per sample in the batch), while workspace buffers are shared.

im2col Workspace Buffer

The cnn_im2col buffer is a temporary workspace used by the im2col + GEMM convolution approach. It holds the column matrix produced by the im2col kernel — a rearranged copy of the input where each column contains all input values needed for one output position.

BufferSizeDescription
cnn_im2col max(Cin × KH × KW × OH × OW) across all conv layers Shared workspace for im2col output. Reused sequentially by each conv layer since layers execute one at a time. Not scaled by batch size — each sample is processed independently.
The im2col buffer is sized to the largest conv layer's column matrix. This avoids allocating separate buffers per layer. The buffer is written by im2col and read by gemm (forward) or gemm_transB / gemm_transA (backward). During backward dInput computation, gemm_transA writes to it and col2im reads from it.
