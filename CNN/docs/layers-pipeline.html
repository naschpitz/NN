Here's a concrete numeric example showing how data flows through a sequence of layers.

Image Classification Pipeline

Example: Image Classification Pipeline

Input: 1 channel, 6×6 image (e.g., grayscale)

\[ \text{Shape: } 1 \times 6 \times 6 \]

Step 1: Conv2D (2 filters, 3×3, stride 1, VALID)

Output shape: \(\text{outH} = (6 + 0 - 3) / 1 + 1 = 4\), \(\text{outW} = 4\)

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: \(2 \times 1 \times 3 \times 3 + 2 = 20\) (18 weights + 2 biases)

Step 2: InstanceNorm

Normalize per channel, scale by \(\gamma\), shift by \(\beta\). Shape unchanged

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: \(2 \times 2 = 4\) learnable (\(\gamma, \beta\)) + 4 running stats

Step 3: ReLU

Element-wise \(\max(0, x)\), shape unchanged

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: 0 (no learnable parameters)

Step 4: MaxPool (2×2, stride 2)

Output shape: \(\text{outH} = (4 - 2) / 2 + 1 = 2\), \(\text{outW} = 2\)

\[ \text{Shape: } 2 \times 2 \times 2 \]

Parameters: 0 (no learnable parameters)

Step 5: Flatten

Reshape to 1D vector: \(2 \times 2 \times 2 = 8\) elements

\[ \text{Shape: } 8 \]

Parameters: 0 (no learnable parameters)

Summary

LayerTypeOutput ShapeParameters
Input—1×6×6—
Conv2D2 filters, 3×32×4×420
InstanceNormNormalization2×4×44 + 4 running
ReLUActivation2×4×40
MaxPool2×2, stride 22×2×20
FlattenReshape80

The 8-element vector can now be fed to a dense layer for classification.

Memory Layout: The CNN library uses NCHW (channels-first) layout. When flattening, elements are ordered as: all values in channel 0, then channel 1, etc. This differs from NHWC (channels-last) used by some frameworks.
