Layer Documentation

Detailed reference for all layer types in the CNN library: Conv2D, ReLU, Pool (Max & Average), and Flatten. Each section covers the mathematical operation, forward pass, backpropagation, and implementation details.

Conv2D Layer

The convolutional layer is the core building block of CNNs. It applies learned filters (kernels) to extract spatial features from the input. Each filter slides across the input, computing dot products to produce feature maps.

Purpose

Conv2D layers learn to detect local patterns (edges, textures, shapes) by applying multiple filters. The layer is parameterized by:

  • Number of filters — how many feature maps to produce
  • Filter size — typically 3×3 or 5×5
  • Stride — how many pixels the filter moves each step
  • Padding — how to handle boundaries (VALID, SAME, FULL)

Forward Pass

For each filter, the forward pass computes:

\[ \text{output}[f, h, w] = \text{bias}[f] + \sum_{c=0}^{C-1} \sum_{kh=0}^{K_h-1} \sum_{kw=0}^{K_w-1} \text{input}[c, h \cdot s_y + kh - p_y, w \cdot s_x + kw - p_x] \times \text{filter}[f, c, kh, kw] \]

Where \(f\) is the filter index, \((h, w)\) is the output position, \(s_y, s_x\) are strides, and \(p_y, p_x\) are padding offsets.

Output Shape Formula

The output spatial dimensions are computed as:

\[ \text{outH} = \left\lfloor \frac{\text{inputH} + 2 \cdot p_y - k_h}{s_y} \right\rfloor + 1 \] \[ \text{outW} = \left\lfloor \frac{\text{inputW} + 2 \cdot p_x - k_w}{s_x} \right\rfloor + 1 \]

The output has \(\text{numFilters}\) channels.

Padding Strategies

The library supports three padding modes:

ModePaddingUse CaseOutput Size
VALID\(p = 0\)Reduce spatial dimensions\(\lfloor (H - k) / s \rfloor + 1\)
SAME\(p = \lfloor k / 2 \rfloor\)Preserve spatial size (stride=1)\(H\) (if stride=1)
FULL\(p = k - 1\)Expand output (transposed conv)\(H + k - 1\)

Backward Pass

Backpropagation computes three gradients simultaneously:

  • dFilters — gradient w.r.t. filter weights: \(\frac{\partial L}{\partial w_{f,c,kh,kw}}\)
  • dBiases — gradient w.r.t. biases: \(\frac{\partial L}{\partial b_f}\)
  • dInput — gradient w.r.t. input: \(\frac{\partial L}{\partial x}\) (passed to previous layer)
Implementation Detail: The backward pass uses transposed convolution to compute dInput, allowing efficient gradient computation through the layer.

Parameters

A Conv2D layer with \(F\) filters, \(C\) input channels, and \(K_h \times K_w\) kernel size has:

\[ \text{Total parameters} = F \times C \times K_h \times K_w + F \]

The first term is filter weights, the second is biases (one per filter).

ReLU Layer

Rectified Linear Unit (ReLU) is a simple but powerful non-linear activation function. It introduces non-linearity, allowing the network to learn complex patterns.

Purpose

ReLU breaks the linearity of the network. Without activation functions, stacking multiple linear layers would be equivalent to a single linear transformation. ReLU is computationally efficient and works well with gradient-based optimization.

Forward Pass

Element-wise operation:

\[ \text{output}[i] = \max(0, \text{input}[i]) \]

Backward Pass

Gradient flows only where input was positive:

\[ \frac{\partial L}{\partial x_i} = \begin{cases} \frac{\partial L}{\partial y_i} & \text{if } x_i > 0 \\ 0 & \text{otherwise} \end{cases} \]
Dying ReLU Problem: If a neuron's input is always negative during training, its gradient becomes zero and it never updates. This is rare in practice but can occur with poor initialization or high learning rates.

Parameters

No learnable parameters. ReLU is a fixed non-linearity.

Pool Layer

Pooling layers reduce spatial dimensions while preserving important features. The library supports Max Pooling and Average Pooling.

Purpose

  • Spatial downsampling — reduce computation and memory
  • Translation invariance — small shifts in input don't change output much
  • Feature aggregation — combine local information

Max Pooling

Takes the maximum value in each pooling window:

\[ \text{output}[c, h, w] = \max_{kh, kw} \text{input}[c, h \cdot s_y + kh, w \cdot s_x + kw] \]

The index of the maximum element is stored for backpropagation.

Average Pooling

Computes the mean of each pooling window:

\[ \text{output}[c, h, w] = \frac{1}{K_h \times K_w} \sum_{kh, kw} \text{input}[c, h \cdot s_y + kh, w \cdot s_x + kw] \]

Backward Pass

Max Pooling: Gradient flows only to the position that had the maximum value:

\[ \frac{\partial L}{\partial x_{max}} = \frac{\partial L}{\partial y}, \quad \frac{\partial L}{\partial x_{other}} = 0 \]

Average Pooling: Gradient is distributed evenly across all positions in the window:

\[ \frac{\partial L}{\partial x_{i,j}} = \frac{1}{K_h \times K_w} \frac{\partial L}{\partial y} \]

Parameters

No learnable parameters. Pooling is a fixed operation.

Flatten Layer

Converts a 3D tensor (channels, height, width) into a 1D vector for input to dense layers.

Purpose

Dense layers expect 1D input. Flatten reshapes the 3D feature maps from convolutional layers into a single vector while preserving the data order (NCHW layout).

Forward Pass

Returns the underlying 1D data vector. Since the CNN library uses NCHW layout internally, the data is already stored contiguously in memory:

\[ \text{output} = \text{input.data} \]

This is a zero-copy operation — no data movement occurs.

Backward Pass

Reshapes the 1D gradient back to the original 3D shape:

\[ \frac{\partial L}{\partial x} = \text{reshape}\left(\frac{\partial L}{\partial y}, \text{inputShape}\right) \]

Parameters

No learnable parameters. Flatten is a pure reshape operation.

Layer Pipeline Example

Here's a concrete numeric example showing how data flows through a sequence of layers:

Example: Image Classification Pipeline

Input: 1 channel, 6×6 image (e.g., grayscale)

\[ \text{Shape: } 1 \times 6 \times 6 \]

Step 1: Conv2D (2 filters, 3×3, stride 1, VALID)

Output shape: \(\text{outH} = (6 + 0 - 3) / 1 + 1 = 4\), \(\text{outW} = 4\)

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: \(2 \times 1 \times 3 \times 3 + 2 = 20\) (18 weights + 2 biases)

Step 2: ReLU

Element-wise \(\max(0, x)\), shape unchanged

\[ \text{Shape: } 2 \times 4 \times 4 \]

Parameters: 0 (no learnable parameters)

Step 3: MaxPool (2×2, stride 2)

Output shape: \(\text{outH} = (4 - 2) / 2 + 1 = 2\), \(\text{outW} = 2\)

\[ \text{Shape: } 2 \times 2 \times 2 \]

Parameters: 0 (no learnable parameters)

Step 4: Flatten

Reshape to 1D vector: \(2 \times 2 \times 2 = 8\) elements

\[ \text{Shape: } 8 \]

Parameters: 0 (no learnable parameters)

Summary

LayerTypeOutput ShapeParameters
Input—1×6×6—
Conv2D2 filters, 3×32×4×420
ReLUActivation2×4×40
MaxPool2×2, stride 22×2×20
FlattenReshape80

The 8-element vector can now be fed to a dense layer for classification.

Memory Layout: The CNN library uses NCHW (channels-first) layout. When flattening, elements are ordered as: all values in channel 0, then channel 1, etc. This differs from NHWC (channels-last) used by some frameworks.
