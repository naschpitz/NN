GlobalAvgPool Layer

Global Average Pooling (GAP) reduces each channel's entire spatial extent (H × W) to a single scalar by computing the spatial mean. If the input has shape \((C, H, W)\), the output has shape \((C, 1, 1)\) — all spatial information is collapsed into one number per channel. This is one of the most important modern CNN techniques, first introduced in the "Network in Network" paper (Lin et al., 2013).

Purpose

Consider a network that produces 128 channels of 28×28 feature maps before the dense layers. Without GlobalAvgPool, flatten produces a vector of \(128 \times 28 \times 28 = 100{,}352\) elements. If the first dense layer has 256 neurons, that's \(100{,}352 \times 256 = 25{,}690{,}112\) parameters — an enormous number that easily overfits on small datasets.

GlobalAvgPool reduces this to just 128 elements (one per channel), so the same dense layer needs only \(128 \times 256 = 32{,}768\) parameters. That's a 784× reduction in parameter count.

  • Massive parameter reduction — eliminates the bottleneck between large spatial feature maps and dense layers. This is the primary reason to use GAP.
  • Built-in spatial invariance — the mean is independent of where a feature appears spatially. An edge detector's average activation is the same whether the edge is in the top-left or bottom-right.
  • Strong regularization — fewer parameters means the model is less prone to memorizing training data. GAP acts as a structural regularizer without needing explicit dropout or weight decay.
  • Input-size agnostic — since GAP produces exactly \(C\) values regardless of spatial dimensions, a trained model can be applied to images of different sizes at inference time. This is impossible with direct flatten, which requires a fixed spatial size.

Propagation

For each channel \(c\), GlobalAvgPool computes the arithmetic mean over all \(H \times W\) spatial positions:

\[ \text{output}[c] = \frac{1}{H \times W} \sum_{h=0}^{H-1} \sum_{w=0}^{W-1} \text{input}[c, h, w] \]

The operation works in-place: it compacts the data to the first \(C\) elements of the tensor's data array and updates the shape to \((C, 1, 1)\). No additional memory allocation is needed.

Numeric Example

Consider a tensor with 2 channels and spatial size 3×3:

Channel 0Channel 1
1.02.03.010.020.030.0
4.05.06.040.050.060.0
7.08.09.070.080.090.0

GlobalAvgPool output:

  • Channel 0: \(\frac{1+2+3+4+5+6+7+8+9}{9} = 5.0\)
  • Channel 1: \(\frac{10+20+30+40+50+60+70+80+90}{9} = 50.0\)

Result: [5.0, 50.0] — just 2 values summarizing the entire spatial content of 18 elements.

Backpropagation

Since the output for each channel is the mean of all spatial positions, the gradient is distributed equally to every spatial element in that channel:

\[ \frac{\partial L}{\partial x_{c,h,w}} = \frac{1}{H \times W} \frac{\partial L}{\partial y_c} \]

Intuitively, if the loss wants channel \(c\)'s mean to increase, every spatial element in that channel gets the same small "nudge" upward. The nudge is scaled by \(\frac{1}{HW}\) because each element contributes equally to the mean.

The gradient tensor is expanded from shape \((C, 1, 1)\) back to the original \((C, H, W)\) shape, with every spatial position in a channel receiving the same gradient value.

Placement in the Network

GlobalAvgPool is typically placed after the last convolutional block (conv + norm + relu) and before flatten:

Conv2D → InstanceNorm → ReLU → GlobalAvgPool → Flatten → Dense

It replaces the traditional approach of using multiple max pools to progressively shrink spatial dimensions. With GAP, the network can maintain larger spatial feature maps deeper into the architecture, preserving fine-grained spatial details until the very end.

Typical placement: GlobalAvgPool is placed after the last convolutional block and before flatten. The number of channels in the last conv block directly determines the input size of the first dense layer.

Parameters

No learnable parameters. GlobalAvgPool is a fixed aggregation operation — it adds zero parameters to the model.
