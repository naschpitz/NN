Normalization

Normalization layers stabilise training by normalising activations per channel. This library supports two modes — Instance Normalization and Batch Normalization — through a single unified Normalization<T> class. The math is identical; only the scope of the statistics differs.

Instance Norm vs Batch Norm

For a feature map with shape \((N, C, H, W)\):

PropertyInstance NormalizationBatch Normalization
Layer type"instancenorm""batchnorm"

Both modes are implemented by the unified Normalization<T> class in CNN_Normalization.hpp/cpp. The LayerType parameter (INSTANCENORM or BATCHNORM) controls the statistics scope. On GPU, the same kernels handle both modes via N and sampleStride parameters.

Forward Pass

For each channel \(c\), compute the mean and variance over \(S\) elements, then normalise and apply learnable scale (\(\gamma\)) and shift (\(\beta\)). The scope \(S\) depends on the mode:

  • InstanceNorm: \(S = H \cdot W\) (one sample's spatial dims)
  • BatchNorm: \(S = N \cdot H \cdot W\) (all samples' spatial dims)

Step 1 — Mean

\[\mu_c = \frac{1}{S} \sum_{i \in \mathcal{S}} X_i\]

Step 2 — Variance

\[\sigma_c^2 = \frac{1}{S} \sum_{i \in \mathcal{S}} (X_i - \mu_c)^2\]

Step 3 — Normalise

\[\hat{X}_i = \frac{X_i - \mu_c}{\sqrt{\sigma_c^2 + \epsilon}}\]

Step 4 — Scale and Shift

\[Y_i = \gamma_c \cdot \hat{X}_i + \beta_c\]

where \(\epsilon\) is a small constant (default \(10^{-5}\)) for numerical stability, and \(\gamma_c\), \(\beta_c\) are learnable parameters initialised to 1 and 0 respectively.

Code: Normalization<T>::propagate(batch, shape, params, config, layerType, training) in CNN_Normalization.cpp. On GPU: norm_compute_mean, norm_compute_var, and norm_normalize kernels in opencl/CNN_Normalization.cpp.cl.

Backward Pass

Given upstream gradient \(\frac{\partial L}{\partial Y}\), we compute gradients for \(\gamma\), \(\beta\), and the input \(X\). Let \(S\) be the normalization scope size (as defined above).

Gamma Gradient

\[\frac{\partial L}{\partial \gamma_c} = \sum_{i \in \mathcal{S}} \frac{\partial L}{\partial Y_i} \cdot \hat{X}_i\]

Beta Gradient

\[\frac{\partial L}{\partial \beta_c} = \sum_{i \in \mathcal{S}} \frac{\partial L}{\partial Y_i}\]

Input Gradient

\[\frac{\partial L}{\partial X_i} = \frac{\gamma_c}{\sqrt{\sigma_c^2 + \epsilon}} \left( \frac{\partial L}{\partial Y_i} - \frac{1}{S} \frac{\partial L}{\partial \beta_c} - \frac{\hat{X}_i}{S} \frac{\partial L}{\partial \gamma_c} \right)\]
Code: Normalization<T>::backpropagate(dBatch, shape, params, config, layerType, ...) in CNN_Normalization.cpp. On GPU: norm_dGammaBeta and norm_dInput kernels.

Running Statistics & Inference

During training, both modes maintain exponential moving averages of the batch statistics:

\[\bar{\mu} \leftarrow (1 - m) \cdot \bar{\mu} + m \cdot \mu_{\text{batch}}\] \[\bar{\sigma}^2 \leftarrow (1 - m) \cdot \bar{\sigma}^2 + m \cdot \sigma^2_{\text{batch}}\]

where \(m\) is the momentum (default 0.1).

At inference time:

  • BatchNorm: uses the running statistics \(\bar{\mu}\), \(\bar{\sigma}^2\) instead of computing batch stats.
  • InstanceNorm: computes per-sample spatial statistics on the fly (same as training). Running stats are maintained but not used during inference.
The \(\gamma\), \(\beta\), running mean, and running variance are stored in the parameters.normalization section of the output JSON.
