Design Patterns

The CNN library employs several key design patterns to achieve flexibility, extensibility, and clean separation of concerns.

🏭 Factory Pattern

Purpose: Transparent CPU/GPU backend selection at runtime.

Implementation: Core<T>::makeCore(const CoreConfig<T>& config) examines config.deviceType and returns either CoreCPU<T> or CoreGPU<T>.

auto core = Core<float>::makeCore(config);
// Returns CoreCPU or CoreGPU transparently
core->train(samples.size(), CNN::makeSampleProvider(samples));  // Works with either backend

Benefit: Client code doesn't need to know which backend is in use. Easy to switch between CPU and GPU without code changes.

🔗 ANN Delegation Pattern

Purpose: Reuse the ANN library for dense layer computation instead of reimplementing.

Implementation: CoreCPU<T> contains a std::unique_ptr<ANN::Core<T>> annCore member. After CNN propagation and flatten, the flattened vector is passed to ANN for dense layer processing.

// In CoreCPU::predict()
Tensor3D<T> cnnOut = propagateCNN(input);
Tensor1D<T> flatInput = Flatten<T>::propagate(cnnOut);
ANN::Input<T> annInput(flatInput.begin(), flatInput.end());
ANN::Output<T> annOutput = annCore->predict(annInput);

Benefit: Avoids code duplication, leverages ANN's optimized dense layer implementation.

📦 Stateless Layer Pattern

Purpose: Layer operations are pure functions with no mutable state.

Implementation: All layer classes (Conv2D, ReLU, Pool, Flatten) contain only static methods. Parameters are passed as arguments, not stored in instance variables.

static Tensor3D<T> Conv2D<T>::propagate(
  const Tensor3D<T>& input,
  const ConvLayerConfig& config,
  const ConvParameters<T>& params);

Benefit: Thread-safe, composable, easy to test. No hidden state to manage.

🎯 Template Pattern

Purpose: Support multiple numeric types (int, float, double) with a single codebase.

Implementation: All classes are templated on T. The template parameter flows through the entire architecture: Core<T>, CoreCPU<T>, Conv2D<T>, etc.

template <typename T>
class Conv2D { ... };

// Instantiate for different types
Conv2D<float>::propagate(...);
Conv2D<double>::propagate(...);

Benefit: Type safety, compile-time specialization, no runtime type checking overhead.
