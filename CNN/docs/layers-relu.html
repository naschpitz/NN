ReLU Layer

Rectified Linear Unit (ReLU) is the most widely used activation function in modern neural networks. Despite its simplicity — it just zeroes out negative values — it is the key ingredient that allows deep networks to learn complex, non-linear relationships in data.

Why Non-Linearity Matters

Without activation functions, a neural network would be entirely linear — no matter how many layers you stack, the output is always a linear function of the input. Mathematically, a sequence of linear transformations \(W_3 \cdot W_2 \cdot W_1 \cdot x\) can always be collapsed into a single matrix multiplication \(W' \cdot x\). This severely limits what the network can learn.

ReLU breaks this linearity by introducing a simple non-linear operation: it passes positive values unchanged and zeroes out negative values. This is enough to allow the network to learn arbitrarily complex decision boundaries when layers are stacked.

Compared to older activation functions like sigmoid or tanh, ReLU has two major advantages:

  • No gradient saturation — sigmoid/tanh have gradients that approach zero for large inputs, causing the "vanishing gradient" problem. ReLU's gradient is exactly 1 for positive inputs, enabling strong gradient flow through deep networks.
  • Computational efficiency — ReLU is just a comparison and selection (max(0, x)), much cheaper than the exponentials required by sigmoid/tanh. This matters when applied millions of times per forward pass.

Propagation

ReLU applies an element-wise operation to every value in the 3D tensor. The spatial shape is unchanged — only individual values are modified:

\[ \text{output}[c, h, w] = \max(0, \text{input}[c, h, w]) \]

Positive values pass through unchanged. Negative values become zero. The operation can be visualized as "clipping" the bottom of the activation distribution:

Numeric Example

Consider a feature map with shape (1 channel, 2×3):

InputAfter ReLU
-2.03.5-0.1→0.03.50.0
1.2-4.00.8→1.20.00.8

Three out of six values were negative and got zeroed out. The positive values (3.5, 1.2, 0.8) pass through unchanged. In practice, roughly 50% of activations are zeroed out by ReLU, creating a sparse representation that helps the network focus on the most relevant features.

Backpropagation

ReLU's gradient is extremely simple: it acts as a binary gate. If the input was positive during the forward pass, the gradient flows through unchanged. If the input was negative (and was zeroed out), the gradient is blocked:

\[ \frac{\partial L}{\partial x_i} = \begin{cases} \frac{\partial L}{\partial y_i} & \text{if } x_i > 0 \\ 0 & \text{otherwise} \end{cases} \]

Using the numeric example above, if the incoming gradient is:

Incoming gradient ∂L/∂yOutgoing gradient ∂L/∂x
0.51.00.3→0.01.00.0
0.70.20.9→0.70.00.9

The gradient at positions where the input was negative (-2.0, -0.1, -4.0) is zeroed out. The gradient at positive positions passes through unchanged.

Dying ReLU Problem: If a neuron's input is always negative across all training samples, its gradient is permanently zero and its weights never update. This "dead neuron" can occur with poor weight initialization or excessively high learning rates that push weights into a region where the output is always negative. In practice, this is rare and can be mitigated by using He initialization (the default in this library) and reasonable learning rates.

Placement in the Network

ReLU is placed after the linear operation (Conv2D) and after normalization (if used). The standard ordering is:

Conv2D → InstanceNorm → ReLU → Pool (optional)

This ordering ensures that:

  • The convolution computes the raw weighted sum (z-values)
  • InstanceNorm normalizes these values to have stable mean and variance
  • ReLU then introduces non-linearity on the normalized values
ReLU after normalization, not before: Placing ReLU after InstanceNorm ensures the normalization sees the full distribution of values (both positive and negative). If ReLU came first, half the values would be zero, creating a skewed distribution that makes normalization less effective.

Parameters

No learnable parameters. ReLU is a fixed, element-wise operation. It adds zero parameters to the model and negligible computation cost.
