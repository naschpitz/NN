Convolution

The 2D convolution operation is the fundamental building block of CNNs. This page covers the forward pass, padding & stride, and backpropagation through convolutional layers.

On this page

  1. 2D Convolution
  2. Padding & Stride
  3. Backpropagation through Conv2D

2D Convolution

Given an input tensor and a set of learnable filters, convolution produces an output by sliding each filter across the input and computing element-wise products.

Mathematical Definition

\[ \text{output}[f, o_h, o_w] = \text{bias}[f] + \sum_{c=0}^{C-1} \sum_{k_h=0}^{K_H-1} \sum_{k_w=0}^{K_W-1} \text{input}[c, o_h \cdot s_y + k_h - p_y, o_w \cdot s_x + k_w - p_x] \cdot \text{filter}[f, c, k_h, k_w] \]

Index Notation

  • f — Filter index (0 to F-1), determines the output channel
  • c — Input channel index (0 to C-1)
  • o_h, o_w — Output spatial position (height, width)
  • k_h, k_w — Filter kernel position (height, width)
  • s_y, s_x — Stride in vertical and horizontal directions
  • p_y, p_x — Padding applied to input (height, width)
  • K_H, K_W — Filter kernel dimensions
  • C — Number of input channels
  • F — Number of filters (output channels)
The input is padded with zeros before convolution. The term o_h · s_y + k_h - p_y computes the actual input index by accounting for stride, kernel position, and padding offset.
Code: Conv2D<T>::propagate(input, config, params) in CNN_Conv2D.cpp implements the full 6-nested-loop convolution. On GPU: the calculate_conv2d kernel in opencl/CNN_Kernels.cpp.cl parallelises over output elements (one work-item per (f, o_h, o_w)).

Padding & Stride

Padding and stride control the spatial dimensions of the output. The output size formula is:

\[ O_H = \left\lfloor \frac{H + 2P_Y - K_H}{S_Y} \right\rfloor + 1 \]

Similarly for width: \(O_W = \left\lfloor \frac{W + 2P_X - K_W}{S_X} \right\rfloor + 1\)

Padding Modes

ModePadding ValueFormulaEffect
VALIDP = 0\(O_H = \lfloor \frac{H - K_H}{S_Y} \rfloor + 1\)Output shrinks; no padding
SAMEP = ⌊K/2⌋\(O_H = \lceil \frac{H}{S_Y} \rceil\)Preserves spatial dims when stride=1
FULLP = K - 1\(O_H = H + K_H - 1\)Output expands; maximum padding
Code: SlidingStrategy::computePadding(kernelSize, strategy) in CNN_SlidingStrategy.hpp returns 0 (VALID), ⌊K/2⌋ (SAME), or K−1 (FULL). The output dimensions are computed inside Conv2D<T>::propagate() using the formula above.

Backpropagation through Conv2D

During backpropagation, gradients flow through the convolution layer in reverse. Given the upstream gradient \(\frac{\partial L}{\partial O[f,o_h,o_w]}\), we compute gradients with respect to filters, biases, and inputs.

Filter Gradient

\[ \frac{\partial L}{\partial W[f,c,k_h,k_w]} = \sum_{o_h} \sum_{o_w} \frac{\partial L}{\partial O[f,o_h,o_w]} \cdot X[c, o_h \cdot s_y + k_h - p_y, o_w \cdot s_x + k_w - p_x] \]
Code: Inside Conv2D<T>::backpropagate() in CNN_Conv2D.cpp — the line dFilters[filterIdx] += dOutVal * input.at(c, ih, iw). On GPU: calculate_dCost_dFilters kernel in opencl/CNN_Kernels.cpp.cl.

Bias Gradient

\[\frac{\partial L}{\partial b[f]} = \sum_{o_h} \sum_{o_w} \frac{\partial L}{\partial O[f,o_h,o_w]}\]
Code: Inside Conv2D<T>::backpropagate() — the line dBiases[f] += dOutVal. On GPU: calculate_dCost_dBiases kernel.

Input Gradient

\[\frac{\partial L}{\partial X[c,i_h,i_w]} = \sum_f \sum_{k_h} \sum_{k_w} \frac{\partial L}{\partial O[f,o_h,o_w]} \cdot W[f,c,k_h,k_w]\]

where \(o_h = \frac{i_h + p_y - k_h}{s_y}\) (only when evenly divisible).

Code: Inside Conv2D<T>::backpropagate() — the line dInput.at(c, ih, iw) += dOutVal * params.filterAt(f, c, kh, kw). On GPU: calculate_dCost_dInput kernel.
The input gradient is essential when stacking multiple convolutional layers, since it propagates the error signal to previous layers.
