Normalization Layers

Instance Normalization normalises activations per channel per sample over spatial dimensions (H × W). It stabilises training, allows higher learning rates, and acts as a mild regulariser. It is typically placed after Conv2D and before ReLU.

Purpose

  • Training stability — normalizing activations prevents them from growing or shrinking across layers
  • Higher learning rates — the normalized distribution is less sensitive to parameter scale
  • Mild regularization — batch statistics introduce noise that can reduce overfitting

Propagation (Training)

For each channel \(c\), compute mean and variance over the spatial dimensions, normalize, then scale and shift with learnable parameters \(\gamma_c\) and \(\beta_c\):

\[ \mu_c = \frac{1}{HW} \sum_{h,w} x_{c,h,w} \qquad \sigma^2_c = \frac{1}{HW} \sum_{h,w} (x_{c,h,w} - \mu_c)^2 \] \[ \hat{x}_{c,h,w} = \frac{x_{c,h,w} - \mu_c}{\sqrt{\sigma^2_c + \epsilon}} \qquad y_{c,h,w} = \gamma_c \, \hat{x}_{c,h,w} + \beta_c \]

Running statistics are updated with exponential moving averages for use during inference:

\[ \bar{\mu}_c \leftarrow (1 - m)\,\bar{\mu}_c + m\,\mu_c \qquad \bar{\sigma}^2_c \leftarrow (1 - m)\,\bar{\sigma}^2_c + m\,\sigma^2_c \]

Propagation (Inference)

During inference, the fixed running statistics replace the batch statistics:

\[ y_{c,h,w} = \gamma_c \, \frac{x_{c,h,w} - \bar{\mu}_c}{\sqrt{\bar{\sigma}^2_c + \epsilon}} + \beta_c \]

Backpropagation

Gradients for the learnable parameters:

\[ \frac{\partial L}{\partial \gamma_c} = \sum_{h,w} \frac{\partial L}{\partial y_{c,h,w}} \, \hat{x}_{c,h,w} \qquad \frac{\partial L}{\partial \beta_c} = \sum_{h,w} \frac{\partial L}{\partial y_{c,h,w}} \]

Input gradient (with \(N = H \times W\)):

\[ \frac{\partial L}{\partial x_{c,i}} = \frac{\gamma_c}{N \sqrt{\sigma^2_c + \epsilon}} \left( N \, \frac{\partial L}{\partial y_{c,i}} - \frac{\partial L}{\partial \beta_c} - \hat{x}_{c,i} \, \frac{\partial L}{\partial \gamma_c} \right) \]

Configuration

ParameterDefaultDescription
epsilon (\(\epsilon\))1×10⁻⁵Numerical stability constant added to variance
momentum (\(m\))0.1EMA factor for running statistics

{ "type": "instancenorm" }

Parameters

An InstanceNorm layer with \(C\) input channels has:

\[ \text{Total parameters} = 2C \text{ (learnable: } \gamma, \beta\text{)} + 2C \text{ (running: } \bar{\mu}, \bar{\sigma}^2\text{)} \]

The output shape is identical to the input shape — InstanceNorm does not change spatial dimensions or channel count.

Typical placement: Conv2D → InstanceNorm → ReLU. Placing InstanceNorm before the activation ensures the normalized distribution is passed to ReLU, which improves gradient flow.
