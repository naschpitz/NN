Memory Layout

The CNN library uses a channels-first (NCHW) memory layout where all spatial elements for a single channel are stored contiguously before the next channel begins.

On this page

  1. NCHW Layout
  2. Tensor Storage
  3. Filter Storage

NCHW Layout

All tensors use NCHW (channels-first) memory ordering:

\[ \text{data}[c \times H \times W + h \times W + w] = \text{tensor}[c, h, w] \]

For a tensor with shape (2 channels, 3 height, 4 width), the memory layout is:

IndexElementDescription
0tensor[0,0,0]Channel 0, Row 0, Col 0
1tensor[0,0,1]Channel 0, Row 0, Col 1
.........
11tensor[0,2,3]Channel 0, Row 2, Col 3
12tensor[1,0,0]Channel 1, Row 0, Col 0
.........
23tensor[1,2,3]Channel 1, Row 2, Col 3
Why NCHW? Channels-first layout enables efficient convolution computation — spatial elements that are convolved together are contiguous in memory, improving cache performance.

Tensor Storage

The Tensor3D<T> class stores data as a flat std::vector<T> paired with a Shape3D struct (channels, height, width). This enables:

  • Zero-cost flattening — the underlying 1D vector can be directly passed to dense layers
  • Efficient resizing — shape changes without reallocating when new size ≤ capacity
  • Direct data access — .data() returns a pointer for raw memory operations (used by GPU)

Filter Storage

Convolution filter weights are stored in a single flat vector with layout:

\[ \text{filters}[f \times C_{in} \times K_h \times K_w + c \times K_h \times K_w + kh \times K_w + kw] \]

Where \(f\) is the filter index, \(c\) is the input channel, and \((kh, kw)\) is the position within the kernel. The filterAt(f, c, h, w) method provides convenient indexed access.
