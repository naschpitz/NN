Class Hierarchy

The CNN library uses a template-based architecture with a factory pattern for CPU/GPU backend selection. All classes are templated on the numeric type T (int, float, or double).

Core Classes

  • Core<T> — Abstract base class defining the interface: predict(), train(), test(). Uses factory pattern via makeCore() static method.
  • CoreCPU<T> — CPU implementation with propagation/backpropagation CNN passes and ANN delegation for dense layers.
  • CoreGPU<T> — GPU implementation managing multiple CoreGPUWorker instances for multi-GPU training.
  • CoreGPUWorker<T> — Per-GPU worker handling OpenCL kernels and local training on a subset of samples.

Stateless Layer Classes

All layer classes contain only static methods — no instance state. They operate on tensors passed as parameters:

  • Conv2D<T> — Convolution with configurable filters, strides, and padding strategies.
  • ReLU<T> — Rectified Linear Unit activation (element-wise max(0, x)).
  • Pool<T> — Max and average pooling with configurable window and stride.
  • Normalization<T> — Instance and Batch normalization with learnable scale/shift.
  • GlobalAvgPool<T> — Global average pooling, reducing each channel to its spatial mean.
  • GlobalDualPool<T> — Concatenated global average and max pooling (doubles channel count).
  • Residual<T> — Skip connections with optional 1×1 projection for channel mismatch.
  • Flatten<T> — Reshapes 3D tensor to 1D vector for dense layer input.
