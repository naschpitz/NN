Optimizers

Optimizers update the network parameters using the computed gradients. Different optimizers use different strategies to control the step size and direction.

On this page

  1. SGD
  2. Adam

SGD (Stochastic Gradient Descent)

The simplest optimizer. Each parameter is updated by subtracting the gradient scaled by the learning rate:

\[\theta_{t+1} = \theta_t - \eta \cdot \frac{\partial L}{\partial \theta_t}\]

where \(\eta\) is the learning rate.

Code: The SGD update is applied in CoreCPU<T>::update() in CNN_CoreCPU.cpp when optimizer.type == SGD. On GPU: update_weights_sgd kernel in opencl/CNN_Kernels.cpp.cl.

Adam (Adaptive Moment Estimation)

Adam maintains per-parameter running averages of the first moment (mean) and second moment (uncentred variance) of the gradients:

Moment Updates

\[m_t = \beta_1 \cdot m_{t-1} + (1 - \beta_1) \cdot g_t\] \[v_t = \beta_2 \cdot v_{t-1} + (1 - \beta_2) \cdot g_t^2\]

Bias Correction

\[\hat{m}_t = \frac{m_t}{1 - \beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1 - \beta_2^t}\]

Parameter Update

\[\theta_{t+1} = \theta_t - \eta \cdot \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}\]

Default hyperparameters: \(\beta_1 = 0.9\), \(\beta_2 = 0.999\), \(\epsilon = 10^{-8}\).

Code: CoreCPU<T>::update() in CNN_CoreCPU.cpp when optimizer.type == ADAM. The Adam state (m, v vectors and timestep t) is stored in Parameters<T>::adamState. On GPU: update_weights_adam kernel in opencl/CNN_Kernels.cpp.cl.
Adam is generally recommended as the default optimizer. It adapts the learning rate per parameter, which helps with sparse gradients and varying gradient magnitudes across layers.
