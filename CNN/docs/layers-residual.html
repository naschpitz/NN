Residual (Skip Connection) Layers

Residual connections add the input of a block directly to its output, enabling gradient flow through very deep networks. The library uses two marker layers — RESIDUAL_START and RESIDUAL_END — to define residual blocks.

Purpose

  • Gradient highway — the skip path lets gradients bypass intermediate layers, preventing vanishing gradients in deep networks
  • Easier optimization — the network only needs to learn the residual (difference from identity), which is often simpler
  • Deeper architectures — enables training networks with dozens or hundreds of layers

Identity Shortcut

When the input and output have the same number of channels, the skip connection is a simple element-wise addition:

\[ \text{output} = \text{block}(x) + x \]

No extra parameters are needed — this is a pure addition.

Projection Shortcut

When the block changes the number of channels (e.g., from 32 to 64), the skip input must be projected to match via a learned 1×1 convolution:

\[ \text{output}[oc, s] = \text{block}(x)[oc, s] + \text{bias}[oc] + \sum_{ic=0}^{C_{in}-1} W[oc, ic] \cdot x[ic, s] \]

The projection is automatically created during initialization when the channel counts differ. The projection weights are He-initialized and biases are zero-initialized.

Spatial dimensions: The skip input and block output must have the same spatial dimensions (H × W). If you need to downsample spatially, place a pool layer outside the residual block (after RESIDUAL_END).

Backpropagation

Identity: The gradient is simply copied to both paths:

\[ \frac{\partial L}{\partial x_{skip}} \mathrel{+}= \frac{\partial L}{\partial y} \]

Projection: Three gradients are computed:

  • dSkip — gradient through the projection: \(\frac{\partial L}{\partial x_{skip}} \mathrel{+}= W^T \cdot \frac{\partial L}{\partial y}\)
  • dW — gradient for projection weights: \(\frac{\partial L}{\partial W} = \frac{\partial L}{\partial y} \cdot x_{skip}^T\)
  • dBias — gradient for projection biases: \(\frac{\partial L}{\partial b} = \sum_s \frac{\partial L}{\partial y}\)

Nesting Rules

  • Every RESIDUAL_START must have a matching RESIDUAL_END
  • Residual blocks can be nested (inner block completes before outer block ends)
  • Spatial dimensions must match between start and end of each block
  • Multiple residual blocks can be chained sequentially

Parameters

Identity shortcut: No learnable parameters.

Projection shortcut: A 1×1 convolution with:

\[ \text{Parameters} = C_{out} \times C_{in} + C_{out} \]

Where \(C_{in}\) is the skip input channels and \(C_{out}\) is the block output channels. The first term is weights, the second is biases.
