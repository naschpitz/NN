GPU Architecture Overview

The CNN library uses a hierarchical architecture to manage GPU computation:

Key Design: CNN layers (Conv2D, ReLU, MaxPool) run on GPU via OpenCL kernels. Dense layers are delegated to the ANN library running on CPU. This hybrid approach leverages GPU strength for spatial operations while keeping dense layer computation on CPU.

Kernel Files

OpenCL kernels are stored in two files, loaded by CoreGPUWorker::allocateBuffers():

FileContents
opencl/CNN_Defines.hpp.cl Type definition: #define TYPE float (configurable for double, int, etc.)
opencl/CNN_Kernels.cpp.cl All propagation, backpropagation, and utility kernels (conv2d, relu, maxpool, accumulate, update, etc.)

Kernels are compiled lazily: only when first needed (predict vs. train modes compile different kernel sets).

GPU Buffer Layout

All GPU buffers are flattened 1D arrays for efficiency. Indices are computed dynamically on the GPU.

Activation & Gradient Buffers

cnn_actvs and cnn_grads store activation values and gradients for all CNN layers, concatenated sequentially.

\[\text{offset}_{\text{layer}} = \sum_{i=0}^{\text{layer}-1} \text{size}_i\]

Index formula: For layer \(l\) with output shape \((C, H, W)\):

\[\text{actvs}[\text{offset}_l + c \cdot H \cdot W + h \cdot W + w]\]

Example: If input is 3×32×32 (size 3072), and first conv outputs 16×30×30 (size 14400), then:

  • Input activations: offset 0, size 3072
  • Conv output activations: offset 3072, size 14400

Filter & Bias Buffers

cnn_filters stores all convolutional layer filters concatenated. For each conv layer with \(F\) filters, \(C_{\text{in}}\) input channels, and kernel size \(K_h \times K_w\):

\[\text{filterSize} = F \cdot C_{\text{in}} \cdot K_h \cdot K_w\]

Index formula:

\[\text{filters}[\text{offset}_{\text{conv}} + f \cdot C_{\text{in}} \cdot K_h \cdot K_w + c \cdot K_h \cdot K_w + k_h \cdot K_w + k_w]\]

cnn_biases stores all biases concatenated (one per filter):

\[\text{biases}[\text{offset}_{\text{conv}} + f]\]

Gradient Accumulation Buffers

cnn_dFilters and cnn_dBiases store per-sample gradients. cnn_accum_dFilters and cnn_accum_dBiases accumulate gradients across samples for parameter updates.

Layout matches filters/biases buffers exactly.

Pool Indices Buffer

cnn_pool_indices stores absolute indices (into actvs buffer) of max values for each max-pool output element. Used during backprop to route gradients to correct input positions.

Size: sum of all max-pool output sizes.

Why buffer allocation lives in CoreGPUWorker, not CoreGPU: Each CoreGPUWorker owns its own OpenCLWrapper::Core, which is bound to a specific GPU device. GPU buffers are device-local — they must be allocated on the device that will use them. This differs from the CPU path, where CoreCPU allocates shared host memory that all CPU workers can reference directly. Since CoreGPU does not own any GPU device context, it cannot allocate device buffers; only the workers can.

Propagation Kernels

calculate_conv2d

Work distribution: One work-item per output element. Total work-items = \(\text{numFilters} \times \text{outH} \times \text{outW}\).

Pseudocode:

kernel void calculate_conv2d(...) {
  gid = get_global_id(0)
  
  // Decompose gid into (f, oh, ow)
  f  = gid / (outH * outW)
  oh = (gid % (outH * outW)) / outW
  ow = gid % outW
  
  sum = biases[bias_offset + f]
  
  for c in 0..inputC:
    for kh in 0..filterH:
      for kw in 0..filterW:
        ih = oh * strideY + kh - padY
        iw = ow * strideX + kw - padX
        
        if ih, iw in bounds:
          input_idx = actv_in_offset + c*inputH*inputW + ih*inputW + iw
          filter_idx = filter_offset + f*inputC*filterH*filterW + c*filterH*filterW + kh*filterW + kw
          sum += actvs[input_idx] * filters[filter_idx]
  
  actvs[actv_out_offset + f*outH*outW + oh*outW + ow] = sum
}

calculate_relu

Work distribution: One work-item per element. Total work-items = layer tensor size.

kernel void calculate_relu(...) {
  gid = get_global_id(0)
  val = actvs[in_offset + gid]
  actvs[out_offset + gid] = (val > 0) ? val : 0
}

calculate_maxpool

Work distribution: One work-item per output element. Total work-items = \(\text{channels} \times \text{outH} \times \text{outW}\).

kernel void calculate_maxpool(...) {
  gid = get_global_id(0)
  
  // Decompose gid into (c, oh, ow)
  c  = gid / (outH * outW)
  oh = (gid % (outH * outW)) / outW
  ow = gid % outW
  
  maxVal = -infinity
  maxIdx = 0
  
  for ph in 0..poolH:
    for pw in 0..poolW:
      ih = oh * strideY + ph
      iw = ow * strideX + pw
      idx = actv_in_offset + c*inputH*inputW + ih*inputW + iw
      
      if actvs[idx] > maxVal:
        maxVal = actvs[idx]
        maxIdx = idx
  
  actvs[actv_out_offset + gid] = maxVal
  pool_indices[pool_idx_offset + gid] = maxIdx  // Store absolute index
}

Backpropagation Kernels

calculate_dCost_dFilters

Work distribution: One work-item per filter weight element. Total work-items = \(\text{numFilters} \times \text{inputC} \times \text{filterH} \times \text{filterW}\).

kernel void calculate_dCost_dFilters(...) {
  gid = get_global_id(0)
  
  // Decompose gid into (f, c, kh, kw)
  f  = gid / (inputC * filterH * filterW)
  c  = (gid % (inputC * filterH * filterW)) / (filterH * filterW)
  kh = ((gid % (inputC * filterH * filterW)) % (filterH * filterW)) / filterW
  kw = gid % filterW
  
  sum = 0
  
  for oh in 0..outH:
    for ow in 0..outW:
      ih = oh * strideY + kh - padY
      iw = ow * strideX + kw - padX
      
      if ih, iw in bounds:
        dOut = grads[grad_out_offset + f*outH*outW + oh*outW + ow]
        inp = actvs[actv_in_offset + c*inputH*inputW + ih*inputW + iw]
        sum += dOut * inp
  
  dFilters[dfilter_offset + gid] = sum
}

calculate_dCost_dBiases

Work distribution: One work-item per filter. Total work-items = numFilters.

kernel void calculate_dCost_dBiases(...) {
  gid = get_global_id(0)  // gid = f
  
  sum = 0
  for oh in 0..outH:
    for ow in 0..outW:
      sum += grads[grad_out_offset + f*outH*outW + oh*outW + ow]
  
  dBiases[dbias_offset + f] = sum
}

calculate_dCost_dInput

Work distribution: One work-item per input element. Total work-items = \(\text{inputC} \times \text{inputH} \times \text{inputW}\).

Uses reverse mapping: for each input position, find which output positions it contributed to.

kernel void calculate_dCost_dInput(...) {
  gid = get_global_id(0)
  
  // Decompose gid into (c, ih, iw)
  c  = gid / (inputH * inputW)
  ih = (gid % (inputH * inputW)) / inputW
  iw = gid % inputW
  
  sum = 0
  
  for f in 0..numFilters:
    for kh in 0..filterH:
      for kw in 0..filterW:
        // Reverse: oh * strideY + kh - padY = ih
        // So oh = (ih + padY - kh) / strideY
        numerator_h = ih + padY - kh
        numerator_w = iw + padX - kw
        
        if numerator_h >= 0 and numerator_w >= 0 and
           numerator_h % strideY == 0 and numerator_w % strideX == 0:
          oh = numerator_h / strideY
          ow = numerator_w / strideX
          
          if oh < outH and ow < outW:
            dOut = grads[grad_out_offset + f*outH*outW + oh*outW + ow]
            filt = filters[filter_offset + f*inputC*filterH*filterW + c*filterH*filterW + kh*filterW + kw]
            sum += dOut * filt
  
  grads[grad_in_offset + gid] = sum
}

calculate_dCost_dRelu

Work distribution: One work-item per element.

kernel void calculate_dCost_dRelu(...) {
  gid = get_global_id(0)
  
  actv = actvs[actv_in_offset + gid]
  dOut = grads[grad_out_offset + gid]
  
  grads[grad_in_offset + gid] = (actv > 0) ? dOut : 0
}

calculate_dCost_dMaxpool

Work distribution: One work-item per pool output element.

kernel void calculate_dCost_dMaxpool(...) {
  gid = get_global_id(0)
  
  dOutVal = grads[grad_out_offset + gid]
  maxIdx = pool_indices[pool_idx_offset + gid]  // Absolute index
  
  // Route gradient to the max position
  grads[maxIdx] += dOutVal
}

Utility Kernels

zero_buffer

Zeros a range of a buffer. Used to clear gradient buffers before backprop.

kernel void zero_buffer(global TYPE* buf, ulong offset, ulong size) {
  gid = get_global_id(0)
  if (gid < size):
    buf[offset + gid] = 0
}

accumulate_gradients

Accumulates per-sample gradients into the accumulator buffer.

kernel void accumulate_gradients(global TYPE* accum, global TYPE* grad, ulong offset, ulong size) {
  gid = get_global_id(0)
  if (gid < size):
    accum[offset + gid] += grad[offset + gid]
}

update_parameters

Updates parameters using accumulated gradients and learning rate.

kernel void update_parameters(global TYPE* params, global TYPE* accum,
                              ulong offset, ulong size, ulong numSamples, float lr) {
  gid = get_global_id(0)
  if (gid < size):
    params[offset + gid] -= lr * (accum[offset + gid] / numSamples)
}

Multi-GPU Training

When multiple GPUs are available, CoreGPU<T> distributes training across workers using QtConcurrent::blockingMap.

Sample Partitioning

Training samples are divided among GPUs. Each worker trains its subset independently, computing local gradients.

CNN Gradient Merging

After each epoch, accumulated filter and bias gradients from all GPUs are merged:

// Pseudocode from CoreGPU::mergeCNNGradients()
totalAccumFilters = zeros(totalFilterSize)
totalAccumBiases = zeros(totalBiasSize)

for each GPU worker:
  workerFilters, workerBiases = worker.readAccumulatedGradients()
  totalAccumFilters += workerFilters
  totalAccumBiases += workerBiases

// Broadcast merged gradients back to all workers
for each GPU worker:
  worker.setAccumulators(totalAccumFilters, totalAccumBiases)

This ensures all GPUs use the same merged gradients for parameter updates.

ANN Parameter Averaging

Dense layer parameters are averaged across all GPUs:

// Pseudocode from CoreGPU::mergeANNParameters()
avgParams = worker[0].getANNParameters()

for i in 1..numGPUs:
  params_i = worker[i].getANNParameters()
  avgParams += params_i

avgParams /= numGPUs

// Broadcast averaged parameters back to all workers
for each GPU worker:
  worker.setANNParameters(avgParams)

Lazy Kernel Compilation

Kernels are compiled only when first needed, not at initialization. This reduces startup time and memory usage.

Predict mode: Compiles only propagation kernels (calculate_conv2d, calculate_relu, calculate_maxpool).
Training mode: Compiles propagation, backpropagation, accumulate, and update kernels.

Flags in CoreGPUWorker track which kernel sets have been compiled:

  • predictKernelsSetup
  • backpropagateKernelsSetup
  • accumulateKernelsSetup
  • updateKernelsSetup
Performance tip: Lazy compilation means the first predict/train call is slower (includes compilation). Subsequent calls reuse compiled kernels.
