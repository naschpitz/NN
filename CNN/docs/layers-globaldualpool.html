GlobalDualPool Layer

Global Dual Pooling (GDP) combines two complementary spatial summaries — Global Average Pooling and Global Max Pooling — by concatenating their outputs. For each channel, it produces two values: the spatial mean and the spatial maximum. This gives the dense layers a richer, more informative representation than either pooling method alone.

Purpose & Intuition

Average pooling and max pooling capture fundamentally different information:

  • Average pooling answers: "How much of this feature is present overall?" — it measures the density of a feature across the spatial extent. If an edge detector fires weakly everywhere, the average is moderate.
  • Max pooling answers: "Is this feature present at all?" — it measures the strongest response regardless of location. If an edge detector fires strongly at just one location, the max is high even if most positions are zero.

By concatenating both, GlobalDualPool gives the dense classifier two complementary "views" of each feature map. The dense layers can then learn to weight these views differently for different features — perhaps using max for rare but distinctive features (e.g., a specific texture) and average for broadly distributed features (e.g., overall brightness).

Research context: This dual-pooling approach is inspired by techniques used in architectures like Squeeze-and-Excitation Networks (SE-Nets) and CBAM, where combining avg and max pooling improves channel attention. GlobalDualPool applies the same principle at the final aggregation stage.

Propagation

For input with \(C\) channels and spatial size \(H \times W\), the output has \(2C\) values arranged as shape \((2C, 1, 1)\). The first \(C\) values are the channel means (avg), and the next \(C\) values are the channel maximums (max):

\[ \text{output}[c] = \frac{1}{HW} \sum_{h=0}^{H-1} \sum_{w=0}^{W-1} \text{input}[c, h, w] \quad \text{for } c \in [0, C) \text{ — average} \] \[ \text{output}[C + c] = \max_{h \in [0,H), \, w \in [0,W)} \text{input}[c, h, w] \quad \text{for } c \in [0, C) \text{ — maximum} \]

The output doubles the channel count: if the input has 128 channels, the output has 256 elements. This must be accounted for in the subsequent flatten and dense layer sizing — the first dense layer must accept \(2C\) inputs.

Internally, the implementation also records the position of the maximum for each channel (the argmax index). This is needed during backpropagation to route the max gradient correctly.

Numeric Example

Consider a tensor with 2 channels and spatial size 2×2:

Channel 0Channel 1
1.04.010.05.0
3.02.08.012.0

GlobalDualPool output (4 values = 2 avg + 2 max):

  • output[0] = avg(ch0) = \(\frac{1+4+3+2}{4} = 2.5\)
  • output[1] = avg(ch1) = \(\frac{10+5+8+12}{4} = 8.75\)
  • output[2] = max(ch0) = \(\max(1,4,3,2) = 4.0\) (at position [0,1])
  • output[3] = max(ch1) = \(\max(10,5,8,12) = 12.0\) (at position [1,1])

Result: [2.5, 8.75, 4.0, 12.0]

Backpropagation

The gradient at each input position combines contributions from both the average path and the max path:

\[ \frac{\partial L}{\partial x_{c,h,w}} = \underbrace{\frac{1}{HW} \frac{\partial L}{\partial y_c^{\text{avg}}}}_{\text{avg gradient (uniform)}} + \underbrace{\frac{\partial L}{\partial y_c^{\text{max}}} \cdot \mathbf{1}[x_{c,h,w} = \max_c]}_{\text{max gradient (sparse)}} \]

Breaking this down:

  • Average gradient: Every spatial position in channel \(c\) receives the same gradient, scaled by \(\frac{1}{HW}\). This is identical to GlobalAvgPool backpropagation.
  • Max gradient: Only the position that held the maximum value receives the gradient. All other positions get zero from the max path. This is identical to max pooling backpropagation.
  • Sum: Both gradients are added together at each position. The max position receives a larger total gradient than other positions.

Using the numeric example above with hypothetical loss gradients \(\frac{\partial L}{\partial y_0^{avg}} = 1.0\) and \(\frac{\partial L}{\partial y_0^{max}} = 2.0\) for channel 0:

PositionAvg gradientMax gradientTotal
[0,0] (val=1)0.2500.25
[0,1] (val=4, max!)0.252.02.25
[1,0] (val=3)0.2500.25
[1,1] (val=2)0.2500.25

GlobalDualPool vs. GlobalAvgPool

AspectGlobalAvgPoolGlobalDualPool
Output size\(C\)\(2C\)
Information capturedFeature density onlyFeature density + presence
Dense layer paramsBaseline2× more (still small)
Best forSimple classificationFine-grained classification
Computation costSlightly lessSlightly more (negligible)
When to choose GlobalDualPool over GlobalAvgPool: Use GlobalDualPool when your classification task requires distinguishing between subtle differences (e.g., different species of birds, different skin conditions). The max component helps the network identify whether a discriminative feature is present anywhere in the image. For simpler tasks, GlobalAvgPool is sufficient and slightly more parameter-efficient.

Parameters

No learnable parameters. GlobalDualPool is a fixed aggregation operation — it adds zero parameters to the model.

JSON Configuration

{ "type": "globaldualpool" }
Output size reminder: If the last convolutional block produces \(C\) channels, GlobalDualPool outputs \(2C\) elements. The first dense layer must accept \(2C\) inputs. This is different from GlobalAvgPool which outputs \(C\) elements.
