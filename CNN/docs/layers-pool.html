Pool Layer

Pooling layers progressively reduce the spatial dimensions of feature maps while retaining the most important information. By shrinking the height and width, pooling reduces the number of computations and parameters in subsequent layers, and also introduces a degree of translation invariance — the network becomes less sensitive to the exact position of features. The library supports two pooling variants: Max Pooling and Average Pooling.

Purpose & Intuition

After a Conv2D + ReLU block detects features (edges, textures, patterns), the feature maps are typically large — often the same spatial size as the input image. Pooling serves three critical purposes:

  • Spatial downsampling — a 2×2 pool with stride 2 cuts each dimension in half, reducing the number of elements by 4×. This dramatically reduces computation and memory in subsequent layers. Without pooling, deeper layers would be prohibitively expensive.
  • Translation invariance — if an edge is detected at pixel (10, 15) versus (11, 15), the 2×2 pool over that region produces the same output. This helps the network recognize objects regardless of small positional shifts.
  • Feature aggregation — pooling combines nearby activations into a single summary value, creating a more abstract representation. Lower layers detect fine details; pooling progressively abstracts these into higher-level concepts.

Output Shape Formula

Pooling preserves the number of channels and reduces spatial dimensions according to:

\[ \text{outH} = \left\lfloor \frac{\text{inH} - K_h}{s_y} \right\rfloor + 1 \qquad \text{outW} = \left\lfloor \frac{\text{inW} - K_w}{s_x} \right\rfloor + 1 \]

For the common case of 2×2 pooling with stride 2, spatial dimensions are exactly halved: a 28×28 input becomes 14×14.

Max Pooling

Max pooling selects the largest value within each pooling window. This preserves the strongest feature activation in each local region:

\[ \text{output}[c, h, w] = \max_{kh \in [0, K_h),\, kw \in [0, K_w)} \text{input}[c, h \cdot s_y + kh, w \cdot s_x + kw] \]

The index (position) of the maximum element within each window is recorded during the forward pass. This information is essential during backpropagation to route the gradient to the correct input position.

Max pooling is the most commonly used variant because it preserves the strongest response — if a filter detected an edge somewhere in a 2×2 region, max pooling keeps that detection regardless of exactly which pixel it was at.

Average Pooling

Average pooling computes the arithmetic mean of all values within each pooling window:

\[ \text{output}[c, h, w] = \frac{1}{K_h \times K_w} \sum_{kh=0}^{K_h-1} \sum_{kw=0}^{K_w-1} \text{input}[c, h \cdot s_y + kh, w \cdot s_x + kw] \]

Average pooling produces a smoother output than max pooling because it considers all values equally. It's particularly useful when you want to preserve the overall activation level rather than just the peak.

Backpropagation

Max Pooling Backward

The gradient flows only to the position that held the maximum value in each window. All other positions receive zero gradient. This is why the forward pass records the max indices — they act as a routing map for the gradient:

\[ \frac{\partial L}{\partial x_{c,h,w}} = \begin{cases} \frac{\partial L}{\partial y_{c,h',w'}} & \text{if } x_{c,h,w} \text{ was the maximum in window } (h', w') \\ 0 & \text{otherwise} \end{cases} \]

Intuitively, during the forward pass, max pooling "chose" one value per window. During backpropagation, the gradient is sent entirely to that chosen value — the other values in the window didn't contribute to the output, so they receive no gradient.

Average Pooling Backward

The gradient is distributed equally to all positions in the window, since every position contributed equally (with weight \(\frac{1}{K_h \times K_w}\)) to the output:

\[ \frac{\partial L}{\partial x_{c,h,w}} = \frac{1}{K_h \times K_w} \frac{\partial L}{\partial y_{c,h',w'}} \]

For a 2×2 average pool, each of the 4 input positions receives \(\frac{1}{4}\) of the output gradient.

Choosing Max vs. Average

AspectMax PoolingAverage Pooling
What it preservesStrongest feature responseOverall feature level
Gradient routingSparse (one position)Uniform (all positions)
Common use caseFeature detection (is edge present?)Texture analysis (average intensity)
Robustness to outliersLow (sensitive to single high value)High (averages out noise)

In practice, max pooling is the default choice for most CNN architectures. Average pooling is more commonly used in the global variant (see GlobalAvgPool).

Parameters

No learnable parameters. Pooling is a fixed operation that adds zero parameters to the model.

JSON Configuration

// Max pooling (2×2, stride 2)
{ "type": "maxpool", "size": 2, "stride": 2 }

// Average pooling (2×2, stride 2)
{ "type": "avgpool", "size": 2, "stride": 2 }
