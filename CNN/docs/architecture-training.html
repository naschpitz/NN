Training Pipeline

Training involves propagation, loss computation, backpropagation, gradient accumulation, and parameter updates. The process repeats for each epoch and sample.

On this page

  1. Training Loop Overview
  2. Detailed Steps

Training Loop Overview

for epoch in [0, numEpochs):
  shuffle(samples)

  for batch of batchSize samples:
    resetCNNAccumulators()
    for sample in batch:
      // 1. Propagate through CNN (store intermediates)
      cnnOut = propagateCNN(sample.input, true, &intermediates, &poolMaxIndices)

      // 2. Flatten and propagate through ANN
      flatInput = Flatten::propagate(cnnOut)
      annOutput = annCore->predict(flatInput)

      // 3. Compute loss (MSE)
      loss = MSE(annOutput, sample.output)

      // 4. Backprop through ANN
      dANNInput = annCore->backpropagate(dLoss)

      // 5. Unflatten and backprop through CNN
      dCNNOut = Flatten::backpropagate(dANNInput)
      backpropagateCNN(dCNNOut, intermediates, poolMaxIndices, dFilters, dBiases)

      // 6. Accumulate gradients
      accumulateCNNGradients(dFilters, dBiases)

    // 7. Update parameters (SGD, per batch)
    updateCNNParameters(batchSize)
    annCore->updateParameters(batchSize)

Detailed Steps

1. Propagation (CNN)

Input flows through CNN layers in order. Intermediates (layer inputs) are stored for backpropagation. Pool layer max indices are recorded for gradient routing.

Tensor3D<T> current = input;
for (const auto& layerConfig : layersConfig.cnnLayers) {
  switch (layerConfig.type) {
    case LayerType::CONV:
      intermediates.push_back(current);
      current = Conv2D<T>::propagate(current, config, params);
      break;
    case LayerType::RELU:
      intermediates.push_back(current);
      current = ReLU<T>::propagate(current);
      break;
    // ... Pool, Flatten
  }
}

2. Propagation (ANN)

Flattened CNN output becomes input to ANN dense layers. ANN handles its own propagation and gradient computation.

Tensor1D<T> flatInput = Flatten<T>::propagate(cnnOut);
ANN::Input<T> annInput(flatInput.begin(), flatInput.end());
ANN::Output<T> annOutput = annCore->predict(annInput);

3. Loss Computation

Mean Squared Error (MSE) between predicted and expected output.

T calculateLoss(const Output<T>& predicted, const Output<T>& expected) {
  T loss = 0;
  for (ulong i = 0; i < predicted.size(); i++) {
    T diff = predicted[i] - expected[i];
    loss += diff * diff;
  }
  return loss / predicted.size();
}

4. Backpropagation (ANN)

ANN computes gradient of loss w.r.t. its input (the flattened CNN output). ANN also updates its own parameters internally.

// ANN backprop returns gradient w.r.t. its input
ANN::Output<T> dANNInput = annCore->backpropagate(dLoss);

5. Backpropagation (CNN)

Gradient flows through CNN layers in reverse order. Each layer computes gradient w.r.t. its input and accumulates gradient w.r.t. its parameters.

// Unflatten gradient from ANN
Tensor3D<T> dCNNOut = Flatten<T>::backpropagate(dANNInput);

// Backprop through CNN layers in reverse
Tensor3D<T> dCurrent = dCNNOut;
for (int i = cnnLayers.size() - 1; i >= 0; i--) {
  switch (cnnLayers[i].type) {
    case LayerType::CONV:
      dCurrent = Conv2D<T>::backpropagate(
        dCurrent, intermediates[i], config, params, dFilters, dBiases);
      break;
    // ... ReLU, Pool
  }
}

6. Gradient Accumulation

Gradients from each sample are accumulated across the epoch. This enables batch-like updates without storing all samples in memory simultaneously.

7. Parameter Update (SGD)

After processing all samples in a mini-batch, accumulated gradients are used to update parameters via Stochastic Gradient Descent.

void updateCNNParameters(ulong numSamples) {
  T learningRate = trainingConfig.learningRate;
  for (ulong i = 0; i < parameters.convParams.size(); i++) {
    for (ulong j = 0; j < parameters.convParams[i].filters.size(); j++) {
      parameters.convParams[i].filters[j] -=
        learningRate * accumDConvFilters[i][j] / numSamples;
    }
  }
}
