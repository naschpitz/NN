2D Convolution

The 2D convolution operation is the fundamental building block of CNNs. Given an input tensor and a set of learnable filters, convolution produces an output by sliding each filter across the input and computing element-wise products.

Mathematical Definition

\[ \text{output}[f, o_h, o_w] = \text{bias}[f] + \sum_{c=0}^{C-1} \sum_{k_h=0}^{K_H-1} \sum_{k_w=0}^{K_W-1} \text{input}[c, o_h \cdot s_y + k_h - p_y, o_w \cdot s_x + k_w - p_x] \cdot \text{filter}[f, c, k_h, k_w] \]

Index Notation

  • f — Filter index (0 to F-1), determines the output channel
  • c — Input channel index (0 to C-1)
  • o_h, o_w — Output spatial position (height, width)
  • k_h, k_w — Filter kernel position (height, width)
  • s_y, s_x — Stride in vertical and horizontal directions
  • p_y, p_x — Padding applied to input (height, width)
  • K_H, K_W — Filter kernel dimensions
  • C — Number of input channels
  • F — Number of filters (output channels)
The input is padded with zeros before convolution. The term o_h · s_y + k_h - p_y computes the actual input index by accounting for stride, kernel position, and padding offset.
Code: Conv2D<T>::predict(input, config, params) in CNN_Conv2D.cpp implements the full 6-nested-loop convolution. On GPU: the calculate_conv2d kernel in opencl/CNN_Kernels.cpp.cl parallelises over output elements (one work-item per (f, o_h, o_w)).

Padding & Stride

Padding and stride control the spatial dimensions of the output. The output size formula is:

\[ O_H = \left\lfloor \frac{H + 2P_Y - K_H}{S_Y} \right\rfloor + 1 \]

Similarly for width: \(O_W = \left\lfloor \frac{W + 2P_X - K_W}{S_X} \right\rfloor + 1\)

Padding Modes

Mode Padding Value Formula Effect
VALID P = 0 \(O_H = \lfloor \frac{H - K_H}{S_Y} \rfloor + 1\) Output shrinks; no padding
SAME P = ⌊K/2⌋ \(O_H = \lceil \frac{H}{S_Y} \rceil\) Preserves spatial dims when stride=1
FULL P = K - 1 \(O_H = H + K_H - 1\) Output expands; maximum padding
When using SAME padding with stride=1, the output spatial dimensions exactly match the input dimensions. This is useful for building deep networks without spatial collapse.
Code: SlidingStrategy::computePadding(kernelSize, strategy) in CNN_SlidingStrategy.hpp returns 0 (VALID), ⌊K/2⌋ (SAME), or K−1 (FULL). The output dimensions are computed inside Conv2D<T>::predict() using the formula above.

Backpropagation through Conv2D

During backpropagation, gradients flow through the convolution layer in reverse. Given the upstream gradient \(\frac{\partial L}{\partial O[f,o_h,o_w]}\), we compute gradients with respect to filters, biases, and inputs.

Filter Gradient

\[ \frac{\partial L}{\partial W[f,c,k_h,k_w]} = \sum_{o_h} \sum_{o_w} \frac{\partial L}{\partial O[f,o_h,o_w]} \cdot X[c, o_h \cdot s_y + k_h - p_y, o_w \cdot s_x + k_w - p_x] \]

This accumulates the product of the upstream gradient and the corresponding input patch for each output position.

Code: Inside Conv2D<T>::backpropagate() in CNN_Conv2D.cpp — the line dFilters[filterIdx] += dOutVal * input.at(c, ih, iw). On GPU: calculate_dCost_dFilters kernel in opencl/CNN_Kernels.cpp.cl (one work-item per filter weight element).

Bias Gradient

\[ \frac{\partial L}{\partial b[f]} = \sum_{o_h} \sum_{o_w} \frac{\partial L}{\partial O[f,o_h,o_w]} \]

The bias gradient is simply the sum of all upstream gradients for that filter across all spatial positions.

Code: Inside Conv2D<T>::backpropagate() — the line dBiases[f] += dOutVal. On GPU: calculate_dCost_dBiases kernel (one work-item per filter).

Input Gradient

\[ \frac{\partial L}{\partial X[c,i_h,i_w]} = \sum_f \sum_{k_h} \sum_{k_w} \frac{\partial L}{\partial O[f,o_h,o_w]} \cdot W[f,c,k_h,k_w] \]

where \(o_h = \frac{i_h + p_y - k_h}{s_y}\) (only when evenly divisible). The input gradient is computed by accumulating contributions from all filters and kernel positions that touch each input element.

Code: Inside Conv2D<T>::backpropagate() — the line dInput.at(c, ih, iw) += dOutVal * params.filterAt(f, c, kh, kw). On GPU: calculate_dCost_dInput kernel (one work-item per input element, reverses the propagation mapping).
The input gradient is essential when stacking multiple convolutional layers, since it propagates the error signal to previous layers. In CoreCPUWorker<T>::backpropagateCNN(), this gradient is passed as dCurrent to the next layer in reverse order.

Pooling Backpropagation

Pooling layers reduce spatial dimensions by aggregating values. During backpropagation, gradients must be routed back to the input elements that contributed to the pooling operation.

Max Pooling

In max pooling, only the maximum value in each pool window contributes to the output. During backpropagation, the gradient flows entirely to the position of the maximum value:

\[ \frac{\partial L}{\partial X[c,i_h,i_w]} = \begin{cases} \frac{\partial L}{\partial O[c,o_h,o_w]} & \text{if } (i_h, i_w) = \text{argmax} \\ 0 & \text{otherwise} \end{cases} \]

Average Pooling

In average pooling, the output is the mean of all values in the pool window. The gradient is distributed equally among all input elements:

\[ \frac{\partial L}{\partial X[c,i_h,i_w]} = \frac{1}{P_H \cdot P_W} \cdot \frac{\partial L}{\partial O[c,o_h,o_w]} \]

where \(P_H\) and \(P_W\) are the pool window dimensions.

Code: Pool<T>::predict(input, config, maxIndices) in CNN_Pool.cpp implements both modes. For max pooling, it stores the flat index of each maximum in maxIndices. Pool<T>::backpropagate() routes the gradient to dInput.data[maxIdx] (max) or distributes dOutVal / count evenly (avg). On GPU: calculate_maxpool records indices, calculate_dCost_dMaxpool routes gradients (both in opencl/CNN_Kernels.cpp.cl).
Max pooling is more commonly used in practice because it preserves sharp features and provides implicit regularization. Average pooling is sometimes used in the final layers for global feature aggregation.

ReLU Backpropagation

The Rectified Linear Unit (ReLU) activation function is defined as \(y = \max(0, x)\). During backpropagation, the gradient is zero for negative inputs and passes through unchanged for positive inputs:

\[ \frac{\partial L}{\partial X_i} = \begin{cases} \frac{\partial L}{\partial Y_i} & \text{if } X_i > 0 \\ 0 & \text{otherwise} \end{cases} \]

This can be written compactly as: \(\frac{\partial L}{\partial X_i} = \mathbb{1}_{X_i > 0} \cdot \frac{\partial L}{\partial Y_i}\), where \(\mathbb{1}\) is the indicator function.

Code: ReLU<T>::predict(input) and ReLU<T>::backpropagate(dOut, input) in CNN_ReLU.cpp. Backpropagation uses the original input to determine where the gradient passes through (input.data[i] > 0). On GPU: calculate_relu and calculate_dCost_dRelu kernels in opencl/CNN_Kernels.cpp.cl.
ReLU is computationally efficient and helps mitigate the vanishing gradient problem compared to sigmoid or tanh activations. The gradient is either 0 or 1, making the backpropagation straightforward.

He Initialization

Proper weight initialization is crucial for training deep networks. He initialization is specifically designed for networks using ReLU activations:

\[ W \sim \mathcal{N}\left(0, \sqrt{\frac{2}{\text{fan\_in}}}\right) \]

where \(\text{fan\_in} = \text{inputChannels} \times \text{filterH} \times \text{filterW}\).

Why He Initialization Works with ReLU

  • Variance preservation: He initialization ensures that the variance of activations remains roughly constant across layers during propagation.
  • Gradient flow: By maintaining consistent activation variance, gradients flow more effectively during backpropagation, avoiding vanishing or exploding gradients.
  • ReLU-specific: The factor of 2 accounts for the fact that ReLU zeros out approximately half of the activations (on average), which would otherwise reduce variance.
Code: CoreCPU<T>::initializeConvParams() in CNN_CoreCPU.cpp computes stddev = sqrt(2.0 / fanIn) where fanIn = inputC × filterH × filterW, then samples from std::normal_distribution. Biases are initialised to zero.
For layers without ReLU (e.g., output layers with softmax), use Xavier/Glorot initialization instead: \(W \sim \mathcal{N}(0, \sqrt{\frac{1}{\text{fan\_in}}})\).

SGD Update Rule

Stochastic Gradient Descent (SGD) is the fundamental optimization algorithm for training neural networks. The weight update rule is:

\[ W^{(t+1)} = W^{(t)} - \eta \cdot \frac{1}{N} \sum_{n=1}^{N} \nabla_W L_n \]

where:

  • \(W^{(t)}\) — Weights at iteration t
  • \(\eta\) — Learning rate (step size)
  • \(N\) — Batch size (number of samples)
  • \(\nabla_W L_n\) — Gradient of loss for sample n with respect to weights

Averaging Over Samples

The gradient is averaged over the batch rather than summed. This provides several benefits:

  • Learning rate independence: The effective step size is independent of batch size.
  • Stability: Averaging reduces noise from individual samples, leading to more stable training.
  • Generalization: Smaller effective steps per sample often lead to better generalization.
Code: CoreCPU<T>::accumulateCNNGradients(dConvFilters, dConvBiases) sums per-sample gradients into epoch accumulators. CoreCPU<T>::updateCNNParameters(numSamples) divides by the sample count and applies the learning rate: params -= lr * (accum / N). On GPU: accumulate_gradients and update_parameters kernels in opencl/CNN_Kernels.cpp.cl.
Modern variants like Adam and RMSprop build on SGD by adaptively scaling the learning rate per parameter, but the core principle of averaging gradients over batches remains.

End-to-End Gradient Flow

In a complete CNN pipeline, gradients flow through multiple layers in reverse. Understanding this flow is essential for debugging training issues.

Typical Architecture

A standard CNN followed by an ANN classifier has this structure:

  1. Input: Raw image tensor \(X \in \mathbb{R}^{B \times C \times H \times W}\)
  2. Conv2D layers: Extract spatial features
  3. ReLU activations: Introduce non-linearity
  4. Pooling layers: Reduce spatial dimensions
  5. Flatten: Convert 4D tensor to 2D matrix
  6. ANN layers: Dense layers for classification
  7. Output: Logits or probabilities

Backpropagation

Gradients flow in reverse order:

\[ \text{Loss} \to \text{ANN output} \to \text{ANN hidden} \to \text{ANN input} \to \text{Unflatten} \to \text{Pool} \to \text{ReLU} \to \text{Conv2D} \to \text{Input} \]

Key Considerations

  • Gradient magnitude: Each layer's backpropagation can amplify or attenuate gradients. Deep networks are prone to vanishing gradients (exponentially small) or exploding gradients (exponentially large).
  • Batch normalization: Often inserted between Conv2D and ReLU to stabilize gradient flow and allow higher learning rates.
  • Skip connections: Residual networks use skip connections to provide direct gradient paths, mitigating vanishing gradients in very deep networks.
  • Gradient clipping: Useful for preventing exploding gradients, especially in recurrent architectures.
The input gradient from the first Conv2D layer is typically discarded since there are no learnable parameters in the input. However, it can be useful for visualizing which input regions are most important for the network's predictions.

Practical Example: 3-Layer CNN

Consider a simple architecture: Conv2D → ReLU → MaxPool → Conv2D → ReLU → MaxPool → Flatten → Dense → Softmax.

During backpropagation:

  1. Softmax backpropagation computes gradient w.r.t. Dense input
  2. Dense backpropagation computes gradient w.r.t. flattened features and updates Dense weights
  3. Unflatten reshapes gradient back to 4D
  4. MaxPool backpropagation routes gradients to max positions
  5. ReLU backpropagation zeros out gradients for negative activations
  6. Conv2D backpropagation computes filter gradients and input gradients
  7. Filter gradients are accumulated and used to update Conv2D weights
  8. Input gradients are discarded (no parameters to update)
Code: Propagation: CoreCPUWorker<T>::propagateCNN(input) iterates Conv2D/ReLU/Pool layers, then Flatten<T>::predict() converts the 3D tensor to a 1D vector, and the ANN's annCore->predict() runs the dense layers. Backpropagation: ANN's annCore->backpropagate() returns the input gradient, Flatten<T>::backpropagate() reshapes it back to 3D, then CoreCPUWorker<T>::backpropagateCNN() iterates CNN layers in reverse, dispatching to Conv2D::backpropagate(), ReLU::backpropagate(), and Pool::backpropagate().
