Conv2D Layer

The convolutional layer is the core building block of CNNs. It applies learned filters (kernels) to extract spatial features from the input. Each filter slides across the input, computing dot products to produce feature maps.

Purpose

Conv2D layers learn to detect local patterns (edges, textures, shapes) by applying multiple filters. The layer is parameterized by:

  • Number of filters — how many feature maps to produce
  • Filter size — typically 3×3 or 5×5
  • Stride — how many pixels the filter moves each step
  • Padding — how to handle boundaries (VALID, SAME, FULL)

Propagation

For each filter, propagation computes:

\[ \text{output}[f, h, w] = \text{bias}[f] + \sum_{c=0}^{C-1} \sum_{kh=0}^{K_h-1} \sum_{kw=0}^{K_w-1} \text{input}[c, h \cdot s_y + kh - p_y, w \cdot s_x + kw - p_x] \times \text{filter}[f, c, kh, kw] \]

Where \(f\) is the filter index, \((h, w)\) is the output position, \(s_y, s_x\) are strides, and \(p_y, p_x\) are padding offsets.

Output Shape Formula

The output spatial dimensions are computed as:

\[ \text{outH} = \left\lfloor \frac{\text{inputH} + 2 \cdot p_y - k_h}{s_y} \right\rfloor + 1 \] \[ \text{outW} = \left\lfloor \frac{\text{inputW} + 2 \cdot p_x - k_w}{s_x} \right\rfloor + 1 \]

The output has \(\text{numFilters}\) channels.

Padding Strategies

The library supports three padding modes:

ModePaddingUse CaseOutput Size
VALID\(p = 0\)Reduce spatial dimensions\(\lfloor (H - k) / s \rfloor + 1\)
SAME\(p = \lfloor k / 2 \rfloor\)Preserve spatial size (stride=1)\(H\) (if stride=1)
FULL\(p = k - 1\)Expand output (transposed conv)\(H + k - 1\)

Backpropagation

Backpropagation computes three gradients simultaneously:

  • dFilters — gradient w.r.t. filter weights: \(\frac{\partial L}{\partial w_{f,c,kh,kw}}\)
  • dBiases — gradient w.r.t. biases: \(\frac{\partial L}{\partial b_f}\)
  • dInput — gradient w.r.t. input: \(\frac{\partial L}{\partial x}\) (passed to previous layer)
Implementation Detail: Backpropagation uses transposed convolution to compute dInput, allowing efficient gradient computation through the layer.

Parameters

A Conv2D layer with \(F\) filters, \(C\) input channels, and \(K_h \times K_w\) kernel size has:

\[ \text{Total parameters} = F \times C \times K_h \times K_w + F \]

The first term is filter weights, the second is biases (one per filter).
