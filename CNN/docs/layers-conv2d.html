Conv2D Layer

The 2D convolutional layer is the fundamental building block of Convolutional Neural Networks. It learns a set of small filters (also called kernels) that slide across the input image, detecting local patterns like edges, textures, and shapes. Each filter produces a feature map — a spatial map showing where in the image the filter's pattern was detected. By stacking many filters, the network builds up increasingly complex and abstract representations of the input.

Purpose & Intuition

Imagine holding a small magnifying glass (the filter) and moving it across a photograph. At each position, you check whether the small region under the glass matches a specific pattern — say, a vertical edge. If it matches well, you record a high value at that position; if not, a low value. The result is a heat map showing where vertical edges appear in the image.

That's exactly what Conv2D does, except:

  • The patterns aren't predefined — they're learned automatically from training data
  • Multiple filters run simultaneously, each detecting a different pattern
  • Filters look at all input channels (e.g., R, G, and B simultaneously), not just one

The layer is parameterized by:

  • Number of filters — how many different patterns to detect (and therefore how many output feature maps to produce). More filters = richer representation but more computation.
  • Filter size — the spatial extent of each filter, typically 3×3 or 5×5 pixels. Smaller filters detect finer-grained patterns; larger filters see more context.
  • Stride — how many pixels the filter moves between positions. Stride 1 examines every position; stride 2 skips every other position, halving the output spatial dimensions.
  • Padding — how to handle the image boundaries where the filter would extend beyond the edge (VALID, SAME, or FULL).

How Filters Detect Patterns

A filter is a small grid of numbers (weights) — typically 3×3. To apply it, the filter is placed over a region of the input image, and at each position we perform an element-wise multiply-and-sum: multiply each filter weight by the corresponding input pixel, then sum all the products. This produces a single output number that indicates how well the input region matches the pattern encoded in the filter.

Key insight: The filter's weight pattern determines what it detects. A horizontal edge filter would have negative weights on top and positive on the bottom. A corner detector would have a more complex weight pattern. In a trained CNN, the network discovers these patterns automatically — the filter weights are adjusted by gradient descent to detect whatever patterns are most useful for the task.

Multi-Channel Convolution

The example above used a single-channel (grayscale) input. Real images have multiple channels (e.g., RGB with 3 channels), and deeper layers in the network have dozens or hundreds of channels. Multi-channel convolution works the same way, but the filter has a separate 2D kernel for each input channel:

  • A filter applied to a 3-channel RGB input is actually three 3×3 grids (one per channel), totaling 27 weights.
  • At each position, the filter computes the multiply-and-sum separately for each channel, then sums across channels to produce a single output value.
  • This means one filter can learn patterns that combine information from all channels — for example, detecting a "red-to-blue transition" by having positive weights in the red channel kernel and negative weights in the blue channel kernel.

When there are multiple filters (say 32), each filter produces its own feature map. The output tensor has 32 channels — one per filter. Each filter has independently learned to detect a different pattern.

Propagation Formula

The full convolution formula puts the multiply-and-sum from the example above into mathematical notation. For each filter \(f\), at each output position \((h, w)\):

\[ \text{output}[f, h, w] = \text{bias}[f] + \sum_{c=0}^{C-1} \sum_{kh=0}^{K_h-1} \sum_{kw=0}^{K_w-1} \text{input}[c, h \cdot s_y + kh - p_y, w \cdot s_x + kw - p_x] \times \text{filter}[f, c, kh, kw] \]

Where \(f\) is the filter index, \((h, w)\) is the output position, \(s_y, s_x\) are strides, and \(p_y, p_x\) are padding offsets.

Output Shape Formula

The output spatial dimensions are computed as:

\[ \text{outH} = \left\lfloor \frac{\text{inputH} + 2 \cdot p_y - k_h}{s_y} \right\rfloor + 1 \] \[ \text{outW} = \left\lfloor \frac{\text{inputW} + 2 \cdot p_x - k_w}{s_x} \right\rfloor + 1 \]

The output has \(\text{numFilters}\) channels.

Why Filters Learn

The vertical edge filter in the example above was hand-designed — we chose the weights [-1, 0, 1] because we knew what pattern we wanted to detect. But in a real CNN, the filter weights start as random numbers. How do they end up detecting useful patterns?

The answer is gradient descent, and the logic works like this:

  1. The network makes a prediction — the input image flows through all the layers and produces an output (e.g., "90% cat, 10% dog").
  2. We measure the error — the loss function compares the prediction to the correct answer. If the image was actually a dog, the loss is high because the network was confidently wrong.
  3. We ask: "which filter weights contributed to this error?" — this is what backpropagation computes. For every single filter weight in every layer, it calculates a gradient: a number telling us how much that weight contributed to the loss, and in which direction.
  4. We adjust the weights to reduce the error — each weight is nudged slightly in the direction that would have reduced the loss. Weights that made the prediction worse are adjusted; weights that didn't contribute much stay roughly the same.

Over thousands of training images, this process converges: filter weights that detect genuinely useful patterns (edges, textures, shapes that help distinguish cats from dogs) are reinforced, while useless weights fade toward values that don't interfere.

Concrete example: how a filter weight gets updated

Suppose filter weight \(w_{0,0}\) (top-left corner of filter 0) currently has value 0.3. During training on one image:

  1. The gradient computation reveals \(\frac{\partial L}{\partial w_{0,0}} = +0.05\) — meaning this weight made the loss slightly higher (positive gradient = increasing loss).
  2. With learning rate 0.01, the weight update is: \(w_{0,0} \leftarrow 0.3 - 0.01 \times 0.05 = 0.2995\)
  3. The weight decreased slightly. If this happens consistently across many images, the weight will keep decreasing, moving toward whatever value minimizes the loss.

Now multiply this by thousands of weights across dozens of filters and thousands of training images. Each weight is independently nudged toward its optimal value. The emergent result is that filters self-organize into meaningful pattern detectors.

What patterns emerge? Research consistently shows that early-layer filters learn simple patterns: horizontal edges, vertical edges, diagonal edges, color blobs, gradients. Deeper layers combine these into textures, then object parts, then whole objects. The network discovers its own feature hierarchy — no one tells it to look for edges first.

Backpropagation

During training, backpropagation through Conv2D computes three separate gradients that are needed for different purposes:

1. Filter weight gradient (dFilters)

How should each filter weight change to reduce the loss? This is computed by correlating the input with the output gradient:

\[ \frac{\partial L}{\partial w_{f,c,kh,kw}} = \sum_{h,w} \frac{\partial L}{\partial y_{f,h,w}} \cdot x_{c, h \cdot s_y + kh, w \cdot s_x + kw} \]

Intuitively, a filter weight should increase if the input and the output gradient have the same sign at the corresponding positions — meaning that increasing this weight would have reduced the loss.

2. Bias gradient (dBiases)

The bias gradient is simply the sum of the output gradient over all spatial positions:

\[ \frac{\partial L}{\partial b_f} = \sum_{h,w} \frac{\partial L}{\partial y_{f,h,w}} \]

3. Input gradient (dInput)

This gradient is passed backward to the previous layer. It answers: "how would changing the input affect the loss?" It is computed using a transposed (full) convolution of the output gradient with the filters:

\[ \frac{\partial L}{\partial x_{c,h,w}} = \sum_{f} \sum_{kh,kw} \frac{\partial L}{\partial y_{f, h', w'}} \cdot w_{f,c,kh,kw} \]

where \((h', w')\) are the output positions that used input position \((h, w)\) during the forward pass.

Implementation detail: The input gradient computation is equivalent to convolving the output gradient with the flipped filters using FULL padding. This elegant relationship between forward convolution and backward gradient computation is one of the mathematical foundations of CNN training.

Weight Initialization

Good weight initialization is critical for training convergence. The library uses He initialization (also called Kaiming initialization), which is designed specifically for layers followed by ReLU:

\[ w \sim \mathcal{N}\left(0, \sqrt{\frac{2}{C_{in} \times K_h \times K_w}}\right) \]

The variance is scaled by the number of inputs to each neuron (fan-in). This ensures that the output variance is approximately 1.0 after ReLU activation, preventing signals from exploding or vanishing as they propagate through many layers.

Biases are initialized to zero.

Parameters

A Conv2D layer with \(F\) filters, \(C\) input channels, and \(K_h \times K_w\) kernel size has:

\[ \text{Total parameters} = F \times C \times K_h \times K_w + F \]

The first term is filter weights (each filter has \(C \times K_h \times K_w\) weights), the second term is biases (one per filter).

For example, a layer with 64 filters, 32 input channels, and 3×3 kernels has \(64 \times 32 \times 3 \times 3 + 64 = 18{,}496\) parameters.
