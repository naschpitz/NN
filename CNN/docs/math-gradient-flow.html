End-to-End Gradient Flow

In a complete CNN pipeline, gradients flow through multiple layers in reverse. Understanding this flow is essential for debugging training issues.

On this page

  1. Typical Architecture
  2. Backpropagation Order
  3. Key Considerations
  4. Practical Example

Typical Architecture

A standard CNN followed by an ANN classifier has this structure:

  1. Input: Raw image tensor \(X \in \mathbb{R}^{C \times H \times W}\)
  2. Conv2D layers: Extract spatial features
  3. Instance Normalization: Stabilise activations per channel
  4. ReLU activations: Introduce non-linearity
  5. Pooling layers: Reduce spatial dimensions
  6. Flatten: Convert 3D tensor to 1D vector
  7. ANN layers: Dense layers for classification
  8. Output: Logits or probabilities

Backpropagation Order

Gradients flow in reverse order:

\[ \text{Loss} \to \text{ANN output} \to \text{ANN hidden} \to \text{ANN input} \to \text{Unflatten} \to \text{Pool} \to \text{ReLU} \to \text{Normalization} \to \text{Conv2D} \to \text{Input} \]

Key Considerations

  • Gradient magnitude: Each layer's backpropagation can amplify or attenuate gradients. Deep networks are prone to vanishing gradients (exponentially small) or exploding gradients (exponentially large).
  • Instance Normalization: Often inserted between Conv2D and ReLU to stabilise gradient flow and allow higher learning rates. Normalises each sample independently over spatial dimensions.
  • Skip connections: Residual networks use skip connections to provide direct gradient paths, mitigating vanishing gradients in very deep networks.
  • Gradient clipping: Useful for preventing exploding gradients, especially in recurrent architectures.
The input gradient from the first Conv2D layer is typically discarded since there are no learnable parameters in the input. However, it can be useful for visualising which input regions are most important for the network's predictions.

Practical Example: 3-Layer CNN

Consider a simple architecture: Conv2D → Normalization → ReLU → MaxPool → Conv2D → Normalization → ReLU → MaxPool → Flatten → Dense → Softmax.

During backpropagation:

  1. Softmax backpropagation computes gradient w.r.t. Dense input
  2. Dense backpropagation computes gradient w.r.t. flattened features and updates Dense weights
  3. Unflatten reshapes gradient back to 3D
  4. MaxPool backpropagation routes gradients to max positions
  5. ReLU backpropagation zeroes out gradients for negative activations
  6. Normalization backpropagation computes dGamma, dBeta, and input gradients
  7. Conv2D backpropagation computes filter gradients and input gradients
  8. Filter and normalization gradients are accumulated and used to update parameters
  9. Input gradients from the first Conv2D are discarded (no parameters to update)
Code: Propagation: CoreCPUWorker<T>::propagateCNN(input) iterates Conv2D/ReLU/Pool/Normalization layers, then Flatten<T>::propagate() converts the 3D tensor to a 1D vector. ANN training step: annCore->trainStep(input, expected) runs the ANN forward pass, backward pass, and gradient accumulation in one call, returning TrainStepResult with the predicted output and input-layer gradients. Backpropagation: The returned inputGradients are reshaped by Flatten<T>::backpropagate() back to 3D, then CoreCPUWorker<T>::backpropagateCNN() iterates CNN layers in reverse, dispatching to Conv2D::backpropagate(), ReLU::backpropagate(), Pool::backpropagate(), and Normalization::backpropagate().
