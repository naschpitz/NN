Quick Start & Build

Prerequisites

RequirementMinimumNotes
C++ CompilerC++17GCC 7+, Clang 5+, MSVC 2017+
CMake3.14
Qt5 or 6Core + Concurrent modules
OpenCL1.2Optional — required only for GPU mode

Clone & Build

# Clone with submodules (ANN + OpenCLWrapper)
git clone --recursive https://github.com/naschpitz/CNN.git
cd CNN

# Build
mkdir build && cd build
cmake ..
make -j$(nproc)

# Run tests
./test_cnn
The CNN repository includes ANN and OpenCLWrapper as git submodules under extern/. The --recursive flag ensures they are cloned automatically.

Use as a CMake Submodule

To use CNN in your own project:

# 1. Add CNN as a submodule
git submodule add https://github.com/naschpitz/CNN.git extern/CNN
git submodule update --init --recursive

# 2. In your CMakeLists.txt:
add_subdirectory(extern/CNN)
target_link_libraries(your_app PRIVATE CNN)

Linking to CNN automatically brings in ANN and OpenCLWrapper as transitive (PUBLIC) dependencies.

Minimal Example

#include "CNN_Core.hpp"

int main() {
    // Configure a simple CNN
    CNN::LayersConfig layers;
    layers.cnnLayers = {
        {CNN::LayerType::CONV, CNN::ConvLayerConfig{8, 3, 3, 1, 1, CNN::SlidingStrategyType::VALID}},
        {CNN::LayerType::RELU, CNN::ReLULayerConfig{}},
        {CNN::LayerType::POOL, CNN::PoolLayerConfig{CNN::PoolTypeEnum::MAX, 2, 2, 2, 2}},
        {CNN::LayerType::FLATTEN, CNN::FlattenLayerConfig{}}
    };
    layers.denseLayers = {
        {64, ANN::ActvFuncType::RELU},
        {10, ANN::ActvFuncType::SIGMOID}
    };

    CNN::CoreConfig<float> config;
    config.deviceType = CNN::DeviceType::CPU;
    config.modeType   = CNN::ModeType::TRAIN;
    config.inputShape = {1, 28, 28};  // 1-channel 28×28 image
    config.layersConfig    = layers;
    config.trainingConfig.numEpochs    = 10;
    config.trainingConfig.batchSize    = 64;
    config.trainingConfig.learningRate = 0.01f;

    auto core = CNN::Core<float>::makeCore(config);

    // Train, predict, test...
    return 0;
}
CNN is a pure computational library — it has no built-in file I/O. The calling application (e.g. NN-CLI) is responsible for loading configuration and saving trained models.
