1. Synopsis

NN-CLI --config <file> [--mode <mode>] [--device <device>]
       [--input <file>] [--input-type <type>]
       [--samples <file>] [--idx-data <file> --idx-labels <file>]
       [--num-epochs <n>]
       [--shuffle-samples <bool>]
       [--output <file>] [--output-type <type>]
       [--log-level <level>]

3. Modes

train

Trains the network on provided samples. Requires --samples or --idx-data/--idx-labels. Outputs a trained model JSON to --output (or auto-generated path). Before training begins, a training summary table is printed showing the full configuration. If validation is enabled (the default), a stratified validation split is created and validation loss is reported during training.

NN-CLI -c config.json -m train -s samples.json
NN-CLI -c config.json -m train --idx-data images.idx3 --idx-labels labels.idx1

predict

Runs inference on one or more inputs in parallel (across threads on CPU, across GPUs on GPU). Requires --input. The config must include pre-trained parameters. Outputs prediction + metadata JSON. Output order matches input order.

NN-CLI -c trained_model.json -m predict -i input.json -o result.json

test

Evaluates loss on a test set. Requires --samples or IDX files. The config must include pre-trained parameters. Prints test metrics (sample count, average loss).

NN-CLI -c trained_model.json -m test -s test_samples.json

5. Network Type Detection

NN-CLI automatically identifies whether a config defines an ANN or CNN:

  • CNN — config contains "inputShape" or "convolutionalLayers"
  • ANN — otherwise (uses "layers" with "numNeurons")

No command-line flag is needed — the detection is transparent.

6. Output Behaviour

Training Output

If --output is not specified, the trained model is saved to:

output/trained_model_<epochs>_<samples>_<loss>.json

The output/ directory is created automatically relative to the input file's location.

Predict Output

When outputType is "vector" (default), the result is JSON with prediction metadata, an outputs array (post-activation values, one per input) and a logits array (pre-activation z of the last layer, one per input). When outputType is "image", the output vector is saved as a PNG/JPEG/BMP image file instead.

{
  "predictMetadata": {
    "startTime": "2024-01-15 10:30:45",
    "endTime": "2024-01-15 10:30:46",
    "durationSeconds": 0.042,
    "durationFormatted": "0.042s",
    "numInputs": 1
  },
  "outputs": [[0.05, 0.92, 0.03]],
  "logits": [[-2.31, 0.62, -2.79]]
}

logits are the raw values that feed the final activation. Use them to compute calibration / out-of-distribution scores (max-logit, logit-norm, free-energy −log Σ exp(zᵢ)) that softmax discards because softmax is shift-invariant.

7. Training Summary

Before training begins, NN-CLI prints a summary table showing the full training configuration at a glance. This helps verify that all settings are correct before committing to a potentially long training run.

The summary includes:

  • Device -- CPU or GPU, with thread/GPU count
  • Network type -- ANN or CNN
  • Input shape -- dimensions of input data
  • Layer counts -- convolutional, dense, and residual layers (CNN) or total layers (ANN)
  • Total parameters -- trainable weight and bias count
  • Training / validation samples -- sample counts after any augmentation and validation split
  • Augmentation -- factor, balance mode, and enabled transforms
  • Class weights -- whether auto class weights are enabled
  • Hyperparameters -- epochs, batch size, learning rate, optimizer, dropout rate
  • Cost function -- type and configuration
  • Shuffle -- whether sample shuffling is enabled

8. Validation Dataset

NN-CLI supports automatic validation dataset splitting. When enabled (the default), a portion of the training samples is held out for validation. After each epoch (or every N epochs, controlled by checkInterval), the model is evaluated on the validation set and the validation loss is reported.

Auto-sizing

When autoSize is true (the default), the validation split ratio is chosen automatically based on the total dataset size:

Dataset SizeValidation Ratio
< 1,000 samples20%
1,000 -- 10,00015%
10,000 -- 100,00010%
100,000 -- 1,000,0002%
> 1,000,0001%

Set autoSize to false and specify size (0.0--1.0) to use a fixed ratio instead.

Stratified splitting

The split is stratified, meaning it preserves the class distribution from the original dataset in both the training and validation subsets. The split is also deterministic -- the same dataset and configuration always produce the same split.

Output

Validation loss is reported after the epoch progress line:

Epoch    5/100 [...] 100.0% - Loss: 0.045678
Validation Loss: 0.052341

See Data Formats for the full validation configuration reference.

9. Training Monitoring

NN-CLI supports training monitoring with early stopping. When enabled, the monitor evaluates metrics at regular intervals and can stop training early if the model is no longer improving or if training has become unstable.

Loss stagnation

Tracks whether the loss is still improving. Training is stopped when the loss does not improve by at least minDelta for patience consecutive check intervals. For example, with checkInterval: 5 and patience: 20, training stops if there is no improvement for 20 consecutive checks (every 5 epochs), i.e., 100 epochs without meaningful progress.

Loss explosion

Detects when training has become unstable. Training is stopped immediately if the current loss exceeds threshold times the best recorded loss, or if the loss becomes NaN. For example, with threshold: 10.0, training stops if the loss grows to more than 10x the best loss seen so far.

Best model

Whenever a new best loss is recorded, the model is saved to output/best_model.json (relative to the input file's location). This file is overwritten each time a new best is found, so it always contains the best model seen during training.

Loss source

When validation is enabled, the monitor uses the validation loss to evaluate metrics. Otherwise, it uses the training loss. This ensures the monitor tracks generalisation performance when a validation set is available.

Output indicators

During training, the epoch line shows a [best] marker when a new best loss is recorded:

Epoch   10/1000 [...] 100.0% - Loss: 0.034521 [best]

When early stopping triggers, a [Monitor] message is printed explaining the reason:

[Monitor] Early stopping: loss stagnation (no improvement for 20 checks)
[Monitor] Early stopping: loss explosion (current: 5.234, best: 0.034, threshold: 10.0x)
[Monitor] Early stopping: loss is NaN

See Data Formats for the full monitoring configuration reference.

10. Progress Bar

During training, NN-CLI displays a real-time progress bar:

Single GPU/CPU:
Epoch    1/1000 [████████████████████████░░░░░░░░░░░░░░░░░░░░░░░░] 50.0% - Loss: 0.234567

Multi-GPU:
Epoch    1/1000 [████████░░░░│████████░░░░│████████░░░░] 50.0% (0: 50% | 1: 50% | 2: 50%)

Updates are throttled: prints on epoch completion, epoch change, every 500 samples, and on the first sample.

11. Error Handling

NN-CLI validates all inputs and provides clear error messages:

ConditionError
Missing --config"Config file is required"
Invalid JSON"Failed to parse config file"
Predict without --input"Input file is required for predict mode"
Both --samples and --idx-data"Cannot use both --samples and --idx-data"
Invalid IDX magic number"Invalid IDX file format"
CNN input size mismatch"Input size does not match inputShape"
Predict/test without parameters"Parameters required for predict/test mode"

All errors print to stderr and exit with code 1.
